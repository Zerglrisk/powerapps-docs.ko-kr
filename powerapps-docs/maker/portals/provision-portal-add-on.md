# <a name="provision-a-portal-using-the-older-portal-add-on"></a>이전 포털 추가 기능을 사용 하 여 포털 프로 비전

이전 포털 추가 기능을 구매한 경우 추가 기능을 사용 하 여 포털을 프로 비전 하려면 **Dynamics 365 관리 센터** 페이지로 이동 하 여 포털을 프로 비전 해야 합니다.

> [!NOTE]
> 포털을 프로 비전 하려면 포털에 대해 선택 된 Common Data Service 환경의 시스템 관리자 또는 시스템 사용자 지정자 역할을 할당 받아야 합니다. 또한 Azure AD에서 응용 프로그램을 만들고 등록 하는 [데 필요한 권한이](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#required-permissions) 있어야 합니다. 필요한 권한이 없는 경우 전역 관리자에 게 문의 하 여 사용 권한을 업데이트 하거나 전역 관리자에 게 포털을 프로 비전 하도록 요청 합니다.

포털을 프로 비전 하려면:

1.  **Dynamics 365 관리 센터** 페이지로 이동한 다음 **응용 프로그램** 탭을 선택 합니다.

2.  **포털 추가 기능**이라는 응용 프로그램 행을 선택 하 고 관리를 선택 **합니다.**

3.  **일반 설정** 섹션에서 포털의 **이름을** 입력 합니다. **이름은** 포털을 식별 하는 데 도움이 되며 나중에 변경할 수 있습니다.

4.  **유형** 필드는 포털 구독의 유형 (평가판 또는 프로덕션)을 나타냅니다. 이는 시스템 필드 이므로 사용자가 변경할 수 없습니다. 이 값은 평가판 구독 인지 또는 유료 구독 인지에 따라 변경 됩니다.

5. 필요에 따라 **포털 개발 상태** 드롭다운 목록에서 포털에 대 한 다음 개발 상태 중 하나를 선택 합니다.

    - 프로토타입에서
    - 개발
    - 테스트할
    - UAT
    - Ttl

    > [!NOTE]
    > - 기존의 프로 비전 된 포털의 경우이 드롭다운 목록은 **포털 세부 정보** 탭에서 사용할 수 있으며 기본적으로 상태는 선택 되지 않습니다.
    > - 이 드롭다운 목록은 프로덕션 유형의 포털 에서만 사용할 수 있습니다.
    > - 이 필드는 Microsoft에서이 포털의 사용 패턴을 이해 하는 데 사용 되며 어떤 기능에도 영향을 주지 않습니다. 개발 수명 주기에 다른 이름을 사용 하는 경우 더 가까운 용도를 선택 하세요. 이는 포털이 프로 비전 되 면 나중에 변경 될 수 있습니다.

5.  포털 **URL** 필드에 포털에 사용할 하위 도메인 이름을 입력 합니다. 영숫자 문자 또는 하이픈 (-)만 사용할 수 있습니다. 다른 문자는 허용 되지 않습니다.

    > [!NOTE]
    > - 프로 비전 된 포털의 URL을 변경 하려면 [포털의 기준 url 변경](admin/change-base-url.md)을 참조 하세요.
    > - 사용자 지정 도메인에 포털을 연결 하려면 [사용자 지정 도메인에 포털 연결](admin/add-custom-domain.md)을 참조 하세요.

6.  **Dynamics 365 인스턴스** 드롭다운 목록에서 포털을 연결 하려는 인스턴스를 선택 합니다. 이렇게 하려면 선택 하는 인스턴스에서 시스템 관리자 또는 시스템 사용자 지정자 역할이 필요 합니다.

7.  **포털 언어 선택** 드롭다운 목록에서 포털에 대 한 기본 언어를 선택 합니다. 사용 가능한 언어는 인스턴스에 설치 된 언어에 따라 달라 집니다. 

    > [!NOTE]
    > 샘플 데이터는 한 가지 언어로만 제공 되므로 기본 언어를 선택 하면 샘플 데이터를 변환 하는 방법도 결정 됩니다. 아랍어와 히브리어는 지원 되지 않으며 목록에 표시 되지 않습니다.

8. 포털 **관리자 선택** 드롭다운 목록에서 포털을 구성 하 고, 사용자 지정 하 고, 유지 관리 하는 사용자를 선택 합니다. 조직에서 시스템 관리자 역할이 있는 모든 사용자가 옵션으로 표시 됩니다. 

9. **포털 대상 그룹** 섹션에서 새 포털을 방문 하는 대상 그룹의 유형을 선택 합니다. 그러면 제공 되는 포털의 옵션이 결정 됩니다. 다음을 선택할 수 있습니다.

    -   Partner    
        -   고객 셀프 서비스 포털
        -   사용자 지정 포털
        -   파트너 포털
        -   파트너 프로젝트 서비스 (선택 사항, 솔루션이 설치 되어 있어야 함)
        -   파트너 필드 서비스 (선택 사항, 솔루션이 설치 되어 있어야 함)
        -   커뮤니티 포털

    -   고객
        -   고객 셀프 서비스 포털
        -   사용자 지정 포털
        -   커뮤니티 포털

    -   직원
        -   직원 셀프 서비스 포털

10. **배포할 포털 선택** 섹션에서 만들려는 포털의 유형을 선택 합니다. 표시 되는 옵션은 선택한 대상 그룹에 따라 달라 집니다.

    > [!div class="mx-imgBorder"]
    > ![포털에 대 한 설정 구성](media/configure-settings-portal.png "포털에 대 한 설정 구성")  

11. **제출**을 선택 하 고 서비스 약관에 동의 합니다.
    > [!div class="mx-imgBorder"]
    > ![서비스 약관](media/terms-of-service.png "서비스 약관")  

서비스 약관에 동의 하면 포털에서 프로 비전을 시작 합니다. 프로 비전은 일반적으로 30 분 정도 걸리며 시스템 부하에 따라 몇 시간이 걸릴 수 있습니다. 응용 프로그램 탭의 포털 *이름은* 프로 비전 하는 동안 *이름*-구성으로 변경 됩니다. 포털 관리 페이지로 다시 이동 하 여 프로 비전이 성공 했는지 확인 합니다.

포털이 프로 비전 되 면 **포털 세부 정보** 페이지에 필요한 세부 정보가 표시 됩니다.

> [!div class="mx-imgBorder"]
> ![포털 정보](media/portal-details-prov.png "포털 정보") 


> [!Note]
> 포털 사용자가 Azure AD 자격 증명을 사용 하 여 처음으로 포털에 로그인 하는 경우 사용자 또는 포털 유형에 관계 없이 모든 사용자에 게 동의 페이지가 표시 됩니다.

## <a name="troubleshoot-provisioning"></a>프로 비전 문제 해결

패키지 설치 프로세스 또는 URL 생성 프로세스에서 오류가 발생 하는 경우가 있습니다. 이러한 경우 프로세스를 다시 시작할 수 있습니다.

이름- *이름*-프로 비전에 대 한 변경 내용을 구성 하지 *못한 경우 프로*비전 프로세스를 다시 시작 해야 합니다.

1. **응용 프로그램** 페이지로 이동 하 여 포털을 선택 합니다.
2. **관리**라는 레이블이 지정 된 파란색 연필 단추를 선택 합니다.
3. 다음 옵션 중 하나를 선택 합니다.

   - **프로 비전 다시 시작**: 이전에 정의 된 구성으로 설치 프로세스를 다시 시작 합니다.

   - **값 변경 및 프로 비전 다시 시작**: 프로 비전 프로세스를 다시 시작 하기 전에 일부 값을 변경할 수 있습니다.

패키지를 설치 하지 못한 경우에는 포털 관리자 페이지가 문제 없이 열리며, 실제 포털 URL로 이동 하면 설정 하는 메시지가 표시 됩니다. 확인 하려면 다음을 수행 합니다.

1. **Dynamics 365 관리 센터** 페이지의 솔루션 관리 페이지로 이동 하 여 패키지 상태가 **설치에 실패**했는지 확인 합니다. 

2. 패키지 상태를 **설치 하지 못한**경우 솔루션 페이지에서 설치를 다시 시도 합니다. 또한 시스템 관리자가 기본 언어를 사용 하 여 솔루션을 설치 하 고 Common Data Service 포털을 설치 해야 하는 언어로 설정 되었는지 확인 해야 합니다.

> [!NOTE]
> 일부 솔루션에는 설치를 위한 필수 구성 요소가 있으므로 필수 구성 요소가 충족 되지 않으면 설치가 실패 합니다. 예를 들어 파트너 포털에 대 한 파트너 필드 서비스를 설치 하려면 파트너 포털 및 필드 서비스 솔루션이 이미 설치 되어 있어야 합니다. 먼저 Partner Field Service를 설치 하려고 하면 설치가 실패 하 고 오류 메시지가 표시 됩니다.
