# <a name="whats-new-in-powerapps"></a>PowerApps의 새로운 기능
알려진 제한 사항에 대한 정보는 [일반적인 문제 및 해결 방법](common-issues-and-resolutions.md)을 참조하세요.

**참고**: PowerApps의 최신 릴리스 버전 번호는 플랫폼에 따라 다릅니다. 플랫폼에서 PowerApps의 최신 릴리스를 사용하려면 Windows 스토어, App Store 또는 Google Play에서 설치합니다.

## <a name="release-20700-20701"></a>릴리스 2.0.700-2.0.701
1. 처음으로 [앱을 저장](save-publish-app.md)하면 기본적으로 2분마다 자동으로 추가적인 변경 사항이 저장됩니다.
2. 식을 작성하지 않고 조건을 설정한 후 PowerApps 캔버스에서 바로 결과를 설계하는 방식으로 손쉽게 조건부 서식에 대한 [규칙을 생성](working-with-rules.md)할 수 있습니다.
3. 컨트롤이 추가되면 표시되는 전체 높이의 데이터 팝업 창을 사용하여 양식, 갤러리 및 데이터 테이블을 더욱 손쉽게 구성할 수 있습니다.
4. 새 앱, 템플릿, 데이터 원본 또는 SharePoint 중 어느 것으로 시작하든 관계없이 앱을 만드는 데 도움을 주는 상황에 맞는 빠른 팁을 확인하세요.

2. **[Split](functions/function-split.md)** 함수를 사용하여 텍스트 문자열을 여러 부분으로 나누려면 구분 기호를 사용합니다.

## <a name="release-20680-20681"></a>릴리스 2.0.680-2.0.681

## <a name="release-20670-20671"></a>릴리스 2.0.670-2.0.671
1. 앱 내에 [Power BI 타일](controls/control-power-bi-tile.md)을 표시합니다.
2. 빈 값을 쉽게 대체하면서 다른 값 그대로 두려면 **[Coalesce](functions/function-isblank-isempty.md)** 함수를 사용합니다.
3. 텍스트 문자열이 다른 텍스트 문자열을 끝내는지 테스트하려면 **[EndsWith](functions/function-startswith.md)** 함수를 사용합니다.
4. 수식에 **[And](functions/operators.md)**, **[Or](functions/operators.md)** 또는 **[Not](functions/operators.md)** 연산자를 사용합니다.

5. [동일한 양식](functions/function-form.md)의 레코드를 표시, 작성 또는 편집합니다.

## <a name="release-20600-20601"></a>릴리스 2.0.600-2.0.601
1. **실험적인 기능: 사용자가 Common Data Service의 관계형 데이터를 보고, 탐색하고, 편집할 수 있는 엔터티 양식 컨트롤로 동적 양식을 추가 합니다.**
   
    자세한 내용은 [엔터티 양식 컨트롤 사용](entity-form-control.md)을 참조하세요.
2. **각 연결 형식에 대한 아이콘으로 데이터 원본을 보다 쉽게 식별합니다.**
   
    PowerApps Studio의 데이터 원본 패널에서 연결을 추가한 경우 SharePoint, OneDrive, YouTube 등의 연결 형식에는 고유 아이콘이 있습니다. 해당 연결을 추가하면 어떤 데이터 원본을 추가했는지 쉽게 기억할 수 있도록 연결 옆에 아이콘이 표시됩니다. 예를 들어 Common Data Service에 대한 연결은 그 이름 및 엔터티의 이름과 함께 해당 연결 형식의 아이콘과 함께 표시됩니다.
3. **색 코드 구문 요소를 통해 보다 쉽게 긴 수식을 읽으세요.**
   
    수식 입력줄에 수식을 입력할 때 서로 다른 구문 요소는 가독성을 높여주고 긴 수식을 쉽게 이해할 수 있도록 여러 색으로 나타납니다. 요소와 해당 색 테이블은 [수식 시작](working-with-formulas.md#formula-syntax)을 참조하세요.
4. **PowerApps 알림 연결을 사용하여 앱 또는 흐름의 푸시 알림을 전송하고 다른 앱을 대상으로 합니다.**
   
    알림을 받고 완전히 사용자 지정된 환경을 위한 대상 앱으로 메시지와 매개 변수를 전달해야 하는 사용자를 선택합니다. 자세한 내용은 [앱에 푸시 알림 추가](https://powerapps.microsoft.com/blog/add-push-notification-to-you-app-and-boost-usage-and-retention/)를 참조하세요.
5. **관리 센터에서 환경의 이름을 바꾸거나 환경을 삭제합니다.**
   
    자세한 내용은 [환경 관리](environments-administration.md)를 참조하세요.
6. **URL을 OpenAPI 정의로 지정하여 사용자 지정 커넥터를 만듭니다.**
   
    자세한 내용은 [사용자 지정 커넥터 등록 및 사용](register-custom-api.md)을 참조하세요.

## <a name="release-20590-20591"></a>릴리스 2.0.590-2.0.591
1. **높이가 다른 항목을 맞추기 위해 크기가 조정된 갤러리에 데이터 표시합니다**.
   
    예를 들어, 길이가 매우 다양한 텍스트(예: 제품 설명)의 경우 가변 높이 갤러리를 추가합니다. 이러한 방법을 사용하여 불필요한 공백 없이 전체 설명을 세로 갤러리에 표시할 수 있습니다.
   
    갤러리에 데이터를 표시하는 방법에 대한 단계별 정보는 [항목의 목록 표시](add-gallery.md)를 참조하세요. 참조 내용은 [갤러리 컨트롤](controls/control-gallery.md)를 참조하세요.
2. **문자열이 지정한 문자 시퀀스로 시작하는지 여부를 결정합니다**.
   
    예를 들어, 사용자가 검색 상자에 몇 글자를 입력하여 필드 시작 부분의 텍스트를 기준으로 항목의 목록을 필터링할 수 있도록 [StartsWith 함수](functions/function-startswith.md)를 사용합니다.
3. **텍스트 문자열이 정규식을 사용하여 지정하는 패턴에 맞는지 확인합니다**.
   
    레코드의 필드(예: 이름, 전자 메일 주소, 전화 번호 또는 미국 사회 보장 번호)와 일치하는지 여부에 따라 true/false 값을 반환하는 [IsMatch 함수](functions/function-ismatch.md)를 사용합니다.
4. **사용자가 Tab 키를 누를 때 컨트롤이 활성화되는 순서를 사용자 지정합니다**.
   
    기본적으로 각 컨트롤의 XY 좌표는 탭 순서를 결정합니다. 다른 순서를 지정하려면 하나 이상 컨트롤의 **TabIndex** 속성을 0보다 큰 정수로 설정합니다. 예를 들어, 값이 1로 설정된 컨트롤의 경우 값이 2 이상으로 설정된 컨트롤 보다 먼저 활성화됩니다. 도구 설명을 추가하고 각 화면의 탭 시퀀스가 최적화하여 앱에 [보다 쉽게 액세스](properties-accessiblity.md)할 수 있습니다.
5. **슬라이더 레일 두께를 사용자 지정합니다**.
   
    사용자는 핸들을 레일의 오른쪽이나 왼쪽으로 끌어서 [슬라이더 컨트롤](controls/control-slider.md) 값을 조정합니다.
6. **Android 장치에서 아래로 살짝 미는 대신 단추를 탭하여 앱 목록을 새로 고칩니다**.
   
    자세한 내용은 참조 [휴대폰 또는 태블릿에서 PowerApps 사용](run-app-client.md)을 참조하세요.
7. **버그 수정 및 성능 개선**.

    Single Sign-On 기능은 SharePoint, Office 365 및 Dynamics CRM과 같은 Azure Active Directory 인증을 사용하는 첫 번째 파티 서비스로 PowerApps 사용자를 자동으로 로그인 합니다.
6. **조회 카드 편집을 추가했습니다.**
   
    이 PowerApps 업데이트를 설치 하는 즉시 **[표시 양식](controls/control-form-detail.md)** 및 **[편집 양식](controls/control-form-detail.md)** 컨트롤은 조히 필드를 다른 표에 표시하지만 이 데이터는 읽기 전용으로 설정됩니다. 다른 변경 내용은 SharePoint 커넥터에서 발생하는 즉시, SharePoint 목록의 데이터에 대해 이 필드를 업데이트할 수 있습니다. (이 기능을 활용하도록 PowerApps를 다시 업데이트하지 않아도 됩니다.)
7. **숫자 카드에 대한 최소/최대 값입니다.**
   
    등급 및 백분율에 대한 것과 같은 숫자 카드는 사용자가 설정한(또는 데이터 원본에 대한 메타데이터에서 설정된) 최소 및 최대값을 반영합니다.
8. **DatePicker 컨트롤에 대한 기본값입니다.**
   
    빈 값이 해당 기본값인 DatePicker 컨트롤을 구성할 수 있습니다.
9. **양식 표시 및 양식 편집 컨트롤에 대한 개선 사항입니다.**
   
    양식 컨트롤은 이제 표시 이름과 이름 필드가 다른 경우 모두를 표시하며, 조회 컨트롤의 표시 이름을 변경할 수 있습니다.

1. **앱 제작을 시작하는 페이지를 간소화했습니다.**

3. **상황별 도움말 링크입니다.**
   
    보다 쉽게 제품에서 도움말을 찾고 커뮤니티를 통해 다른 사용자와 연결할 수 있도록 만들었습니다.
4. **새 [편집 양식](controls/control-form-detail.md) 및 [표시 양식](controls/control-form-detail.md) 컨트롤입니다. 이로 인해 데이터에서 더 나은 앱을 생성할 수 있습니다.**
   
    데이터를 통해 앱을 만들면 보다 쉽게 작성됩니다.  이러한 컨트롤 및 해당 관련된 카드와 함수를 사용하여, **[Patch](functions/function-patch.md)**, **[Validate](functions/function-validate.md)** 및 기타 하위 수준 함수를 수동으로 호출하지 않아도 데이터를 보고 편집하도록 기본 앱을 작성할 수 있습니다.
   
    이러한 컨트롤은 필드를 켜거나 끄고 각 필드에 대한 카드를 변경할 수 있는 새 구성 창을 제공합니다.
   
    자세한 내용은 [데이터 양식 이해](working-with-forms.md)을 확인합니다.
5. **[필터](functions/function-filter-lookup.md) 및 [정렬](functions/function-sort.md)의 데이터 원본 위임은 성능을 개선시키고 수많은 레코드를 지원합니다.**
   
    지금까지 모든 필터 및 정렬 작업은 장치에서 처리되며, 이는 시간이 많이 소요되고 모든 데이터 페치를 대역폭 초킹합니다.  이 릴리스의 특정 상황에서 이 작업은 데이터 원본으로 위임될 수 있으며, 앱이 보다 즉각적으로 응답하여 다수의 레코드를 처리할 수 있게 합니다.  자세한 내용은 [필터](functions/function-filter-lookup.md) 및 [정렬](functions/function-sort.md) 페이지에서 읽을 수 있습니다.
   
    위임은 특정 상황으로 제한됩니다. 데이터에서 만든 앱은 보증되지 않습니다.  자세한 내용은 이 문서의 뒷부분에 나와 있는 [일반적인 문제 및 해결 방법](common-issues-and-resolutions.md)을 참조하세요.  위임을 사용할 수 있는 시나리오를 확장합니다.
   
    이 새로운 기능을 활용하려면, **[갤러리](controls/control-gallery.md)** 컨트롤은 이제 페이징을 지원합니다.  사용자가 데이터 목록의 끝에 도달하면 컨트롤은 데이터 소스에서 더 원활하게 많이 페치됩니다.
6. **[PDF 뷰어](controls/control-pdf-viewer.md) 컨트롤에서 검색하고 스크롤합니다.**
   
    이제 PDF 뷰어 컨트롤은 문서에서 전체 텍스트 검색을 지원하고 페이지 간 부드러운 스크롤을 추가했습니다. 더 이상 페이지 단위로 이동하지 않아도 됩니다.

    한 번 클릭하거나 탭하여(연결 삭제 및 다시 추가 대신) 메타데이터를 업데이트합니다.
