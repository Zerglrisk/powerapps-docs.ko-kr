# <a name="configure-web-form-metadata-for-portals"></a>포털의 웹 양식 메타데이터 구성

웹 양식 메타데이터에는 기본 엔터티 양식 편집 기능으로는 불가능한 양식 필드 기능 강화 또는 재정의를 위한 추가 동작 수정 논리를 포함합니다.

## <a name="add-a-new-record"></a>새 레코드 추가

1. 수정하려는 필드가 있는 웹 양식 단계에서 **관련 항목** > **메타데이터**로 이동합니다. 

2. **새 웹 양식 메타데이터**를 선택합니다.

## <a name="web-form-metadata-properties"></a>웹 양식 메타데이터 속성

다음 특성은 양식 요소에 추가 스타일 및 기능을 제공합니다.

| 이름          | 설명                                                                                                                                                                                                                                                                                                                                          |
|---------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 웹 양식 단계 | 웹 양식 메타데이터 레코드와 관련된 웹 양식 단계입니다.                                                                                                                                                                                                                                                                                      |
| 유형          | 사용 가능한 옵션은 다음과 같습니다.<ul><li>특성</li><li>섹션</li><li>탭</li></ul>유형 값으로 특성을 선택하면 연관된 단계에 렌더링된 현재 폼에서 필드를 수정하기 위한 적절한 옵션이 표시됩니다. 유형 값으로 섹션을 선택하면 양식에서 섹션을 수정하는 데 사용할 수 있는 옵션이 표시됩니다. 유형 값으로 탭을 선택하면 양식에서 탭을 수정하는 데 사용할 수 있는 옵션이 표시됩니다.  |

## <a name="web-form-metadata-type--attribute"></a>웹 양식 메타데이터 형식 = 특성

다음 속성은 선택한 유형이 '특성'일 때 표시됩니다.


|          이름          |                                                                                                                                                    설명                                                                                                                                                     |
|------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 특성 논리 이름 |                                                                                                                              수정할 특성 필드의 논리적 이름입니다.                                                                                                                               |
|         레이블          | 엔터티에서 특성에 할당된 기본 레이블을 이 입력에서 지정한 텍스트로 바꿉니다. Common Data Service 환경을 위해 각 언어 팩을 설치하고 활성화하면 필드에 관련 언어로 메시지를 입력할 수 있습니다. |

### <a name="prepopulate-field"></a>필드 미리 채움

다음 옵션은 양식의 필드에 기본값을 제공합니다.


|         이름         |                                                                                                                                                                                                                                                                설명                                                                                                                                                                                                                                                                 |
|----------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 기본값 무시 |                                                                              지정된 특성 필드의 기본값을 무시합니다. 예 및 아니요 라디오 단추로 렌더링 되는 두 개의 옵션 필드인 특성에 유용합니다. 기본적으로 예 또는 아니요 값을 자동으로 할당하기 때문에 이 옵션을 사용하면 미리정의 된 응답 없이 예/아니요 질문을 표시할 수 있습니다.                                                                               |
|         타입         | (다음 버전 중 하나)<ul><li>값</li><li>오늘 날짜</li><li>현재 사용자의 연락처</li></ul>값을 선택하려면 양식이 로드될 때 해당 필드에 할당될 **값** 필드에 값을 지정해야 합니다. 오늘 날짜를 선택하면 특성 필드에 현재 날짜와 시간이 할당됩니다. 현재 사용자의 연락처를 선택하려면 현재 사용자의 연락처 레코드에서 검색되고 지정된 특성 필드에서 설정되는 연락처 엔터티의 특성인 **From 특성**이 필요합니다. |
|        값         |                                                                                                                                                                                                                                        양식이 로드될 때 해당 필드에 할당할 값입니다.                                                                                                                                                                                                                                        |
|    From 특성    |                                                                                                                                                                                             현재 포털 사용자의 레코드에서 검색되고 양식이 로드될 때 해당 필드에 할당되는 연락처 엔터티의 특성입니다.                                                                                                                                                                                             |

### <a name="set-value-on-save"></a>저장 시 값 설정

다음 옵션은 양식이 저장될 때 설정할 값을 지정합니다.

| 이름              | 설명                                                                                                                                                                                                                                                                                                                                                                                                                                |
|-------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 저장 시 값 설정 | 값이 예이면 **값** 필드에 제공된 입력 사항을 사용하여 해당 특성에 할당되는 것을 의미합니다. <br>**참고**: 고유 식별자를 제외한 모든 특성 형식이 지원됩니다.       |
| 타입              | (다음 버전 중 하나)<ul><li>값</li><li>  오늘 날짜</li><li>현재 사용자의 연락처</li></ul>값을 선택하려면 양식이 로드될 때 해당 필드에 할당될 **값** 필드에 값을 지정해야 합니다. 오늘 날짜를 선택하면 특성 필드에 현재 날짜와 시간이 할당됩니다. 현재 사용자의 연락처를 선택하려면 현재 사용자의 연락처 레코드에서 검색되고 지정된 특성 필드에서 설정되는 연락처 엔터티의 특성인 **From 특성**이 필요합니다.  |
| 값             | 양식이 저장될 때 해당 특성에 할당되는 값입니다.<br>두 개 옵션(부울) 필드의 경우 true 또는 false를 사용합니다.<br>옵션 집합 필드의 경우 옵션에 정수 값을 사용합니다.<br>조회(EntityReference) 필드의 경우 GUID를 사용합니다.<br>**참고**: 특성이 양식에도 있는 경우 이 값이 사용자의 값을 덮어쓰게 됩니다.|
| From 특성    | 현재 포털 사용자의 레코드에서 검색되고 저장 중 해당 필드에 할당되는 연락처 엔터티의 특성입니다. |

### <a name="validation"></a>유효성 검사

다음 섹션에는 다양한 유효성 검사 매개 변수 및 오류 메시지를 수정하는 속성이 포함되어 있습니다.

Common Data Service 환경을 위해 각 언어 팩을 설치하고 활성화하면 필드에 관련 언어로 메시지를 입력할 수 있습니다.

| 이름                                        | 설명                                                                                                                                                                                                                                                      |
|---------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 유효성 검사 오류 메시지                    | 필드의 기본 유효성 검사 오류 메시지를 다시 정의합니다.                                                                                                                                                                                                    |
| 정규식                          | 필드 유효성 검사에 추가될 정규식입니다.                                                                                                                                                                                                          |
| 정규식 유효성 검사 오류 메시지 | 유효성 검사를 마친 정규식이 실패할 경우 표시할 유효성 검사 오류 메시지입니다.                                                                                                                                                                               |
| 필수 필드                           | 값을 포함시키려면 특성 필드가 필수인지 확인하십시오.                                                                                                                                                                                                   |
| 필수 필드 유효성 검사 오류 메시지     | 필드에 값이 포함되어 있지 않으면 기본 필수 필드 오류 메시지를 다시 정의합니다.                                                                                                                                                                        |
| 범위 유효성 검사 오류 메시지              | 필드 값이 정수, 10진수, 부동 소수점 숫자 또는 통화 유형인 엔터티 특성에 지정된 적절한 최소값 및 최대값에서 벗어날 경우 표시되는 기본 범위 유효성 검사 오류 메시지를 다시 정의합니다. |
| 지리적 위치 유효성 검사 오류 메시지         | 특성이 한 줄 텍스트이고 지정된 컨트롤 스타일이 지리적 위치 조회 유효성 검사기로서 한 줄 텍스트인 경우에 적용할 수 있으며, 입력 유효성 검사에 실패할 경우 표시되는 기본 오류 메시지를 다시 정의합니다. |
| 상수 합계 유효성 검사 오류 메시지       | 특성이 정수 유형이고 지정된 컨트롤 스타일이 정수를 상수 합계로 그룹화인 경우 적용할 수 있으며 입력된 유효성 검사에 실패할 경우 표시되는 기본 오류 메시지를 다시 정의합니다.                    |
| 여러 선택 항목 유효성 검사 오류 메시지    | 특성이 두 개 옵션 유형이고 지정된 컨트롤 스타일이 여러 선택 항목인 경우 적용할 수 있으며 입력 유효성 검사에 실패할 경우 표시되는 기본 오류 메시지를 다시 정의합니다.                                         |
| 동률 없는 순위 순서 유효성 검사 오류 메시지 | 특성이 정수 유형이고 지정된 컨트롤 스타일이 정수를 동률 없는 순위 순서로 그룹화인 경우 적용할 수 있으며 입력 유효성 검사에 실패할 경우 표시되는 기본 오류 메시지를 다시 정의합니다.              |
