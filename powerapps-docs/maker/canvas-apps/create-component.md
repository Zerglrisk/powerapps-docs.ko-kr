# <a name="create-a-component-for-canvas-apps"></a>캔버스 앱에 대 한 구성 요소 만들기

> [!IMPORTANT]
> 이 기능은 여전히 실험적이 며 기본적으로 비활성화 됩니다. 자세한 내용은 [실험적 및 미리 보기 기능](working-with-experimental.md)합니다.

구성 요소는 캔버스 앱에 대 한 재사용 가능한 구성 요소는 앱 작성자는 앱에서 또는 앱에서 사용 하 여 사용자 지정 컨트롤을 만들 수 있도록 합니다. 사용자 지정 속성 등의 고급 기능 구성 요소에서 복잡 한 기능을 사용 하도록 설정 합니다. 이 문서에서는 구성 요소 개념 및 몇 가지 예를 소개 합니다.

구성 요소는 비슷한 컨트롤 패턴을 포함 하는 더 큰 앱 구축에 유용 합니다. 구성 요소 정의 업데이트 하는 경우 앱의 모든 인스턴스에 변경 내용을 반영 합니다. 또한 복사한 오버 헤드를 복제 하는 컨트롤에 붙여 넣는 하지 때문에 하나 이상의 구성 요소를 사용 하 여 성능을 개선할 수 있습니다. 구성 요소는 또한 공동 개발을 용이 하 게 하 고 조직에서 모양과 느낌을 표준화 합니다.

## <a name="prerequisite"></a>필수 조건

열기는 **앱 설정** 화면에서 **고급 설정**, 기능을 활성화 뿐만 아니라 되도록 **향상 된 앱 렌더링** 도 사용 합니다.

## <a name="component-canvas"></a>구성 요소 캔버스

구성 요소를 만들 수 있습니다는 **구성 요소** 메뉴에서를 **삽입** 탭 또는, 왼쪽된 탐색 모음에서 다음 그래픽 보여 줍니다. 이 목록은 생성 시간을 기준으로 정렬 된 앱에서 정의 된 구성 요소를 보여줍니다.

![구성 요소 목록 보기](./media/create-component/list-view.png)

빈 캔버스를 표시, 어떤 방법을 선택 하든지에 관계 없이 구성 요소 정의의 일부로 컨트롤을 추가할 수 있습니다. 캔버스에서 구성 요소를 편집 하는 경우에 다른 앱 화면 및 다른 앱에서 동일한 구성 요소의 인스턴스를 업데이트 하겠습니다.

화면을 선택 하면 왼쪽된 탐색 모음에서 기존 구성 요소 목록에서 구성 요소를 선택할 수 있습니다 또는 **구성 요소** 메뉴의 **삽입** 탭 합니다. 구성 요소를 선택 하면 컨트롤을 삽입 하는 것 처럼 화면에 해당 구성 요소의 인스턴스에 삽입 합니다.

## <a name="scope"></a>범위

구성 요소 인터페이스와 속성을 사용 하 여 캡슐화 된 블랙 박스도 간주 합니다. 구성 요소를 외부에서 구성 요소에서 컨트롤에 액세스할 수 없습니다 및 외부 구성 요소 내에서 구성 요소에서 아무 것도 참조할 수 없습니다. 시도 하면 오류가 나타납니다. 간단 하 고 통합 되어 범위 제한을 구성 요소의 데이터 계약을 유지 하 고 원활 하 게 구성 요소 정의 업데이트를 앱에서 특히 있도록 키를 누릅니다. 하나 이상의 사용자 지정 속성을 만들어 구성 요소의 데이터 계약을 업데이트할 수 있습니다.

## <a name="variables"></a>변수

구성 요소를 지원 하지 않습니다 합니다 **UpdateContext** 있지만 함수를 만들고 사용 하 여 구성 요소에서 변수를 업데이트할 수 있습니다 합니다 **설정** 함수입니다. 이러한 변수의 범위 구성 요소를로 제한 되지만 액세스할 수 있습니다에서 외부 구성 요소 사용자 지정 출력 속성을 활용 하 여 합니다.

## <a name="import-and-export"></a>가져오기 및 내보내기

구성 요소를 내보낼 경우에 다른 앱을 가져올 수 있는 로컬 파일을 만듭니다. 동일한 구성 요소의 수정 된 버전을 포함 하는 앱을 수정 된 버전을 대체 하거나 가져오기를 취소 여부를 결정할 수 묻는 메시지가 나타납니다. 이 문서의 작성 시점 현재 클라우드 구성 요소를 저장 하거나 환경 내에서 공유할 수 없습니다.

![가져오기 및 내보내기](./media/create-component/import.png)

## <a name="custom-properties"></a>사용자 지정 속성

구성 요소는 입력된 값을 수신 하 고 하나 이상의 사용자 지정 속성을 만드는 경우 데이터를 내보낼 수 있습니다. 이러한 시나리오는 고급 및 수식 이해 해야 할 계약 바인딩 및 합니다.

입력 속성이 구성 요소 구성 요소에서 사용할 데이터를 수신 하는 방법을 보여 줍니다. 입력된 속성에 표시 된 **속성** 구성 요소의 인스턴스를 선택한 경우 오른쪽 창의 탭 합니다. 다른 컨트롤의 표준 속성을 구성 하는 것 처럼 식 또는 수식을 사용 하 여 입력된 속성을 구성할 수 있습니다. 다른 컨트롤 입력 속성을 같은 합니다 **기본** 의 속성을 **텍스트 입력** 컨트롤입니다.

출력 속성 데이터 또는 구성 요소 상태를 내보낼 수 있습니다. 예를 들어를 **선택한** 속성을 **갤러리** 컨트롤 출력 속성입니다. 출력 속성을 만들 때 구성 요소 상태를 참조 하는 다른 컨트롤 수를 확인할 수 있습니다.

이 연습을 더 이상 이러한 개념을 설명 합니다.

    표시 이름을 표시 합니다 **속성** 구성 요소를 선택 하면 오른쪽 창의 탭 합니다. 이 속성의 용도 이해 하는 다른 작성자 및 설명이 포함 된 표시 이름을 도와줍니다. **설명을** 에서이 속성의 표시 이름을 가리키면 도구 설명에 표시 되는 **속성** 탭 합니다.

1. 에 **데이터 형식** 목록에서 **테이블**를 선택한 후 **만들기**합니다.

## <a name="known-limitations"></a>알려진 제한 사항

- 이 문서의 작성 시점 현재 사용 하지 않도록 폼 및 데이터 테이블은 데이터 원본 구성 요소를 사용 하 여 저장 되지 않습니다. 
- 구성 요소에서 변수를 만든 경우 해당 변수는 구성 요소에만 범위가 지정 됩니다 및 앱 변수를 사용 하 여 표시 되지 않습니다.
- PowerApps 구성 요소에서 컬렉션을 지원 하지 않습니다.
- 구성 요소는 갤러리, 폼 또는 데이터 카드를 삽입할 수 없습니다.
- 구성 요소를 마스터 인스턴스가 마스터 및 앱에 범위가 지정 된 로컬 사용 하는 것입니다. 마스터 인스턴스를 변경 하면 앱 내에서 요소의 복사본에만 변경 내용이 반영 됩니다. 구성 요소 라이브러리를 다시 가져올 때까지 다른 앱의 복사본은 그대로 유지 됩니다. 해당 앱의 모든 마스터 인스턴스에 자동으로 검색 되며 업데이트 합니다.
- 구성 요소를 가져와야 하는 경우에 미디어 파일을 패키지로 만들 수 없습니다.
