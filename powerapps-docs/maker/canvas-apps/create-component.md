# <a name="create-a-component-for-canvas-apps"></a>캔버스 앱에 대한 구성 요소 만들기

> [!IMPORTANT]
> 이 기능은 여전히 실험적이며 기본적으로 비활성화됩니다. 자세한 내용은 [실험적 및 미리 보기 기능](working-with-experimental.md)을 확인합니다.

구성 요소는 캔버스 앱에 대한 재사용 가능한 구성 요소이며 앱 작성자는 앱에서 또는 여러 앱에서 사용할 사용자 지정 컨트롤을 만들 수 있습니다. 사용자 지정 속성 등의 고급 기능은 구성 요소에서 복잡한 기능을 사용하도록 설정합니다. 이 문서에서는 구성 요소 개념 및 몇 가지 예를 소개합니다.

구성 요소는 비슷한 컨트롤 패턴을 포함하는 더 큰 앱 구축에 유용합니다. 구성 요소 정의를 업데이트하는 경우 앱의 모든 인스턴스는 변경 내용을 반영합니다. 또한 컨트롤을 복사하고 붙여 넣지(오버헤드가 배가됨) 않기 때문에 하나 이상의 구성 요소를 사용하여 성능을 개선할 수 있습니다. 구성 요소는 또한 공동 개발을 용이하게 하고 조직에서 모양과 느낌을 표준화합니다.

## <a name="prerequisite"></a>필수 조건

**앱 설정** 화면을 열어, **고급 설정**을 선택하고, 기능 활성화 뿐만 아니라 **향상된 앱 렌더링**도 활성화합니다.

## <a name="component-canvas"></a>구성 요소 캔버스

**삽입** 탭의 **구성 요소** 메뉴에서 또는 다음 그래픽에서 보여주는 것처럼 왼쪽 탐색 모음에서 구성 요소를 만들 수 있습니다. 이 목록은 생성 시간을 기준으로 정렬된, 앱에서 정의된 구성 요소를 보여줍니다.

![구성 요소 목록 보기](./media/create-component/list-view.png)

어떤 방법을 선택하든지에 관계 없이 빈 캔버스가 나타나며, 구성 요소 정의의 일부로 컨트롤을 추가할 수 있습니다. 캔버스에서 구성 요소를 편집하는 경우, 다른 앱 화면 및 다른 앱에서 동일한 구성 요소의 인스턴스가 업데이트됩니다.

화면을 선택하면 왼쪽 탐색 모음에서 기존 구성 요소 목록에서 구성 요소를 선택하거나 **삽입** 탭의 **구성 요소** 메뉴에서 선택할 수 있습니다. 구성 요소를 선택하면 컨트롤을 삽입하는 것처럼 화면에 해당 구성 요소의 인스턴스를 삽입합니다.

## <a name="scope"></a>범위

인터페이스와 속성이 있는 캡슐화된 블랙 박스로 구성 요소를 생각해 봅시다. 구성 요소 외부에서 구성 요소의 컨트롤에 액세스할 수 없으며 구성 요소 내에서 구성 요소 외부의 어떤 것도 참조할 수 없습니다. 시도하면 오류가 나타납니다. 범위 제한은 구성 요소의 데이터 계약을 간단하고 일관성있게 유지합니다. 특히 여러 앱에서의 원활한 구성 요소 정의 업데이트를 가능하게 합니다. 하나 이상의 사용자 지정 속성을 만들어 구성 요소의 데이터 계약을 업데이트할 수 있습니다.

## <a name="variables"></a>변수

구성 요소는 **UpdateContext** 함수를 지원하지 않지만, **Set** 함수를 사용하여 구성 요소의 변수를 만들고 업데이트할 수 있습니다. 이러한 변수의 범위는 구성 요소로 제한되지만, 사용자 지정 출력 속성을 활용하여 구성 요소 외부에서 액세스할 수 있습니다.

## <a name="import-and-export"></a>가져오기 및 내보내기

구성 요소를 내보낼 경우, 다른 앱으로 가져올 수 있는 로컬 파일을 만듭니다. 앱이 동일한 구성 요소의 수정된 버전을 포함하는 경우, 수정된 버전을 대체하거나 가져오기를 취소할지 여부를 결정할 수 있는 메시지가 나타납니다. 이 문서의 작성 시점에서는 클라우드로 구성 요소를 저장하거나 환경 내에서 공유할 수 없습니다.

![가져오기 및 내보내기](./media/create-component/import.png)

## <a name="custom-properties"></a>사용자 지정 속성

구성 요소는 입력된 값을 수신하고 하나 이상의 사용자 지정 속성을 만드는 경우 데이터를 내보낼 수 있습니다. 이러한 시나리오는 고급이며 수식과 계약 바인딩을 이해해야 합니다.

입력 속성은 구성 요소 안에서 구성 요소에서 사용할 데이터를 수신하는 방법을 보여줍니다. 구성 요소의 인스턴스를 선택한 경우 오른쪽 창의 **속성** 탭에 입력 속성이 표시됩니다. 다른 컨트롤의 표준 속성을 구성하는 것처럼 식 또는 수식을 사용하여 입력 속성을 구성할 수 있습니다. 다른 컨트롤은 **텍스트 입력** 컨트롤의 **Default** 속성 같은 입력 속성을 가지고 있습니다.

출력 속성은 데이터 또는 구성 요소 상태를 내보낼 수 있습니다. 예를 들어 **갤러리** 컨트롤의 **Selected** 속성이 출력 속성입니다. 출력 속성을 만들 때 어떤 컨트롤이 구성 요소 상태를 참조할 수 있는지를 결정할 수 있습니다.

이 연습은 계속하여 이러한 개념을 설명합니다.

## <a name="create-an-example-component"></a>예제 구성 요소 만들기

이 예제에서는, 다음 그래픽과 유사한 메뉴 구성 요소를 생성하여 여러 화면, 앱 또는 둘 모두에서 사용하고 텍스트를 변경할 수 있습니다.

![최종 갤러리](./media/create-component/menu-instance.png)

1. PowerApps Studio에서, 빈 앱을 만듭니다.

1. 왼쪽 탐색 모음에서 구성 요소 목록을 열어 **새 구성 요소**를 선택합니다.

    ![구성 요소 목록](./media/create-component/component-list.png)

1. 새 구성 요소를 가리켜 줄임표(...)를 선택, **이름 바꾸기**를 선택하여 **MenuComponent**를 입력하거나 붙여 넣습니다.

1. 오른쪽 창에서 구성 요소의 너비를 **150**, 높이를 **250**으로 설정하고 **새 사용자 지정 속성**을 선택합니다.

    ![새 속성](./media/create-component/new-property.png)

1. **표시 이름**, **속성 이름**, 및 **설명** 상자에 **Items**를 입력하거나 붙여 넣습니다.

    ![표시 이름, 속성 이름, 설명 상자](./media/create-component/property-names.png)

    속성 이름을 지정하는 경우, 식을 작성할 때 이름으로 구성 요소를 참조하기 때문에 공백을 포함하지 않도록 합니다(예를 들어 **ComponentName.PropertyName**).

    구성 요소를 선택하면 오른쪽 창의 **속성** 탭에 표시 이름이 나타납니다. 설명적인 표시 이름은 다른 작성자 및 사용자가 이 속성의 용도를 이해하는 데 도움이 됩니다. **속성** 탭에서 표시 이름을 가리키면 도구 설명으로 **설명**이 표시됩니다.

1. **데이터 형식** 목록에서 **테이블**를 선택한 후 **만들기**를 선택합니다.

    ![속성의 데이터 형식](./media/create-component/property-data-type.png)

    **Items** 속성은 지정한 데이터 형식을 기반으로 기본 값으로 설정되지만 필요에 맞는 값으로 설정할 수 있습니다. **테이블** 또는 **레코드**의 데이터 형식으로 지정한 경우, 구성 요소에 입력하려는 데이터 스키마와 일치하도록 **Items** 속성의 값을 변경할 수 있습니다. 이 경우 문자열 목록으로 변경할 수 있습니다.

    오른쪽 창의 **속성** 탭에서 속성의 이름을 선택하는 경우 수식 입력줄에서 속성의 값을 설정할 수 있습니다.

    ![속성 탭에서 사용자 지정 입력된 속성](./media/create-component/properties-tab.png)

    다음 그래픽에서처럼, 오른쪽 창의 **고급** 탭에서 속성 값을 편집할 수도 있습니다. 

1. 구성 요소의 **Items** 속성을 다음 수식으로 설정합니다.

    ```powerapps-dot
    Table({Item:"SampleText"})
    ```

    ![수식](./media/create-component/set-component-items.png)

1. 구성 요소에서, 세로(비어 있음) **갤러리** 컨트롤을 삽입합니다.

1. 속성 목록에 **Items** 속성이 표시되는지(기본으로 표시됨) 확인한 다음, 해당 속성의 값을 다음 식으로 설정합니다.

    ```powerapps-dot
    MenuComponent.Items
    ```

    이러한 방식으로 **갤러리** 컨트롤의 **Items** 속성을 읽을 수 있으며 이는 구성 요소의 **Items** 입력 속성에 따라 달라지게 됩니다.

1. **갤러리** 컨트롤의 **BorderThickness** 속성을 **1**로 설정하고 **TemplateSize** 속성을 **50**으로 설정합니다.

1. **갤러리** 컨트롤의 템플릿에 **레이블** 컨트롤을 추가합니다.

    ![레이블을 추가 하 고 갤러리의 테두리를 설정 합니다.](./media/create-component/add-label.png)

다음으로, 화면에 구성 요소를 추가하고 표시할 구성 요소에 대한 문자열의 테이블을 지정합니다.

1. 왼쪽 탐색 모음에서 화면 목록을 선택하고 기본 화면을 선택합니다.

    ![기본 화면](./media/create-component/default-screen.png)

1. **삽입** 탭에서, **구성 요소** 메뉴를 열고, **MenuComponent**를 선택합니다.

    ![삽입](./media/create-component/insert.png)

    새 구성 요소의 이름은 기본으로 **MenuComponent_1**이 됩니다.

1. **MenuComponent_1**의 **Items** 속성을 다음 수식으로 설정합니다.

    ```powerapps-dot
    Table({Item:"Home"}, {Item:"Admin"}, {Item:"About"}, {Item:"Help"})
    ```

    이 인스턴스는 다음 그래픽과 비슷하지만 각 인스턴스의 다른 속성과 텍스트를 사용자 지정할 수 있습니다.

    ![최종 갤러리](./media/create-component/menu-instance.png)

지금까지 구성 요소를 생성하고 앱에 추가했습니다. 그런 다음 메뉴에서 사용자가 선택한 항목을 반영하는 출력 속성을 만들어야 합니다.

1. 구성 요소 목록을 열고 **MenuComponent**를 선택합니다.

1. 오른쪽 창에서 **속성** 탭을 선택한 후 **새 사용자 지정 속성**을 선택합니다.

1. **표시 이름**, **속성 이름** 및 **설명** 상자에 **Selected**를 입력하거나 붙여 넣습니다.

1. **속성 형식** 아래에서 **출력**을 선택한 후 **만들기**를 선택합니다.

1. **고급** 탭에서 **Selected** 속성 값을 다음 식으로 설정합니다. 필요한 경우 갤러리 이름의 숫자를 조정합니다.

    ```powerapps-dot
    Gallery1.Selected.Item
    ```

    ![고급 창](./media/create-component/advance.png)

1. 앱의 기본 화면에서 레이블을 추가하고 해당 **Text** 속성을 다음 식으로 설정합니다. 필요한 경우 구성 요소 이름의 숫자를 조정합니다.

    ```powerapps-dot
    MenuComponent_1.Selected
    ```

    **MenuComponent_1**은 구성 요소 정의의 이름이 아니라 인스턴스 기본 이름입니다. 인스턴스 이름은 변경할 수 있습니다.

1. Alt 키를 누른 채 메뉴에서 각 항목을 선택합니다.

    **레이블** 컨트롤이 가장 최근에 선택한 메뉴 항목을 반영합니다.

## <a name="known-limitations"></a>알려진 제한 사항

- 이 문서의 작성 시점에서 데이터 원본은 구성 요소를 사용하여 저장되지 않으며, 따라서 폼 및 데이터 테이블은 비활성화됩니다.
- 구성 요소에서 변수를 만든 경우 해당 변수는 구성 요소로만 범위가 지정되며 앱 변수를 사용하여 표시되지 않습니다.
- PowerApps는 구성 요소에서 컬렉션을 지원하지 않습니다.
- 구성 요소를 갤러리, 폼 또는 데이터 카드에 삽입할 수 없습니다.
- 구성 요소의 마스터 인스턴스가 로컬 마스터이며 앱으로 범위가 지정됩니다. 마스터 인스턴스를 변경하면 앱 내의 구성 요소의 복사본에만 변경 내용이 반영됩니다. 구성 요소 라이브러리를 다시 가져오지 않으면 다른 앱의 복사본은 그대로 유지됩니다. 해당 앱의 모든 마스터 인스턴스는 자동으로 검색되며 업데이트됩니다.
- 구성 요소를 가져와야 하는 경우에 미디어 파일을 패키지로 만들 수 없습니다.
