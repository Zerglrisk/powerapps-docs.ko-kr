### <a name="size-limits"></a>크기 제한

텍스트 문자열과 Uri, 이러한 데이터 형식 길이에 미리 설정 된 제한이 있습니다.

이러한 데이터 형식은 또한 참조 하는 이진 데이터를에 크기에 제한이 없습니다. 예를 들어, 이제으로 참조 되는 카메라 컨트롤을 통해 캡처한 이미지 **"appres:..."** 장치의 카메라를 모을 수 있는 크고 높은 해상도로 될 수 있습니다. 해상도, 프레임 속도 및 미디어 파일의 다른 특성 데이터 형식에 따라 제한 되지 않지만 재생 하 고 미디어 캡처에 대 한 특정 컨트롤 자체 제한이 있습니다.

그러나 모든 데이터 크기는 앱에서 사용 가능한 메모리 양을 적용 됩니다. 일반적으로 데스크톱 컴퓨터에서 실행 되는 브라우저 100mb 이상 데이터를 지원 합니다. 그러나 휴대폰과 같은 장치에서 사용 가능한 메모리 양을 범위에서 일반적으로 훨씬 더 낮은 않을 30-70 메가바이트입니다. 앱은 이러한 한도 내에서 실행 여부를 확인 하려면 실행 해야 하는 모든 장치에서 일반적인 시나리오를 테스트 합니다.

모범 사례로, 필요에 따라 오래만 메모리에 데이터를 보관 합니다. 다음을 할 수 있습니다; 즉시 데이터베이스에 이미지 업로드 앱의 사용자가 요청할 경우에 이미지를 다운로드 합니다.

- **표준 시간대 독립**: 앱은 동일한 방식으로 이러한 값을 표시 하 고 앱 사용자 지정 표준 시간대에 관계 없이 동일 하 게 합니다. 동일한 순간 일본의 사용자와 캐나다에서 사용자에 게 동일한 방식으로 나타납니다. 앱 작성자가 앱을 다른 표준 시간대에 실행 하지 않을 전체 간단 이기 때문에 이러한 값을 사용 합니다.

### <a name="mixing-date-and-time-information"></a>날짜 및 시간 정보를 혼합합니다.

**날짜**, **시간**, 및 **DateTime** 이름이 다른 하지만 같은 날짜 및 시간 정보를 보유 합니다. 

A **날짜** 값 일반적으로, 시간 정보를 포함할 수 자정입니다. A **시간** 값 날짜 정보는 일반적으로 수행할 수 1970 년 1 월 1 일입니다. Common Data Service에는 또한 시간 정보를 저장 한 **날짜만** 필드는 하지만 날짜 정보만 기본적으로 표시 합니다. 마찬가지로, 캔버스 앱은 경우에 따라 이러한 데이터 형식을 기본 형식 확인 및 컨트롤 간의 구분 합니다.

추가한 날짜 및 시간 값을 직접 뺀 및 다른 표준 시간대 변환 혼란 스러운 결과가 발생할 수 있으므로 권장 되지 않습니다. 사용 하 여는 **값** 시간 (밀리초)을 먼저 날짜/시간 값을 변환 하 고 앱 사용자의 시간대를 고려 하려면 함수를 사용 하 여 또는 [ **DateAdd** ](function-dateadd-datediff.md) 및 [ **DateDiff** ](function-dateadd-datediff.md) 함수를 추가 하 여 이러한 값 중 하나를 뺍니다.
