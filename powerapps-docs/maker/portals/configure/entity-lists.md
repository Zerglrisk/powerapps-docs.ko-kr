# <a name="about-entity-lists"></a>엔터티 목록 정보

엔터티 목록은 개발자가 포털에서 그리드를 표시할 필요 없이 레코드 목록을 렌더링 하는 웹 페이지를 추가 하는 데 사용 하는 데이터 기반 구성입니다. 엔터티 목록을 사용 하면 포털에 표시할 레코드를 노출할 수 있습니다.

표는 정렬을 지원 하며 레코드 수가 지정 된 페이지 크기 보다 큰 경우 페이지를 매길 수 있습니다. **자세히 보기에 대 한 웹 페이지가** 지정 된 경우 각 레코드는 해당 페이지에 대 한 링크를 포함 하 고 레코드의 ID는 Id 쿼리 문자열 매개 변수 이름과 함께 쿼리 문자열에 추가 됩니다. 엔터티 목록에도 여러 뷰가 지원 됩니다. 둘 이상의 보기가 지정 된 경우 사용자가 다양 한 보기 간을 전환할 수 있도록 드롭다운 목록이 렌더링 됩니다.

현재 포털 사용자, 현재 포털 사용자의 부모 고객 계정 및 현재 포털 웹 사이트를 기준으로 데이터를 필터링 할 수도 있습니다. 두 필터 조건 **포털 사용자 특성** 및 **계정 특성**에 대 한 값이 있으면 포털에서는 사용자가 자신의 (내) 데이터 또는 부모 고객 계정 데이터를 볼 수 있도록 드롭다운 목록을 렌더링 합니다.

## <a name="add-an-entity-list-to-your-portal"></a>포털에 엔터티 목록 추가

엔터티 목록에는 웹 페이지에 대 한 관계 및 포털 내의 레코드 목록 초기화를 제어 하는 다양 한 속성이 포함 되어 있습니다. 웹 페이지와의 관계를 통해 웹 사이트 내에서 지정 된 페이지 노드에 대 한 목록 정의를 동적으로 검색할 수 있습니다. 기존 엔터티 뷰를 보거나 새 엔터티 뷰를 만들려면 **포털** > **엔터티 목록**으로 이동 합니다.

> [!Note]
> - 사이트 내에서 볼 수 있도록 목록을 지정 된 웹 사이트의 웹 페이지와 연결 해야 합니다.
> - 다중 선택 옵션 집합은 엔터티 목록에서 지원 되지 않습니다.

엔터티 목록과 연결 된 웹 페이지는 맨 왼쪽 메뉴의 **관련** 탐색 링크에 나열 된 **웹 페이지** 링크를 선택 하 여 볼 수 있습니다. 엔터티 목록을 만들 때 첫 번째 단계는 포털에서 목록을 렌더링 하려는 엔터티를 선택 하는 것입니다. 그런 다음 렌더링할 모델 기반 앱 뷰를 하나 이상 선택 합니다.

웹 페이지를 만들거나 편집할 때 웹 페이지 양식에 제공 된 조회 필드에서 엔터티 목록을 지정할 수 있습니다. 페이지 템플릿은 일반적으로 페이지 템플릿이 되지만, 마스터 템플릿에 엔터티 목록을 렌더링 해야 하는지 여부를 결정 하는 데 필요한 논리가 포함 되어 있으므로 콘텐츠에 대해 디자인 된 여러 다른 템플릿 중 하나일 수 있습니다.

## <a name="add-custom-javascript"></a>사용자 지정 Javascript 추가

양식의 옵션 탭에는 사용자 지정 [!INCLUDE[pn-javascript](../../../includes/pn-javascript.md)]입력할 수 있는 텍스트 영역이 포함 되어 있습니다. 페이지에 jQuery library가 포함 되어 있는 경우 여기 에서도 사용할 수 있습니다. 스크립트 블록은 페이지의 닫는 form 태그 바로 앞에 추가 됩니다.

![사용자 지정 JavaScript 예제](../media/custom-javascript-example.png "사용자 지정 JavaScript 예제")  

목록에서 데이터를 비동기적으로 가져오고, 완료 되 면 사용자 지정 [!INCLUDE[pn-javascript](../../../includes/pn-javascript.md)]에서 수신 하 고 표의 항목을 사용 하 여 작업을 수행할 수 있는 이벤트 `loaded`를 트리거합니다. 다음 코드는 간단한 예제입니다.
```
$(document).ready(function (){
$(".entitylist.entity-grid").on("loaded", function () {
$(this).children(".view-grid").find("tr").each(function (){
// do something with each row
$(this).css("background-color", "yellow");
});
});
}); 
```

특정 특성 필드를 찾아 값을 가져와 값의 렌더링을 수정할 수 있습니다. 다음 코드는 `accountnumber` 특성의 값을 포함 하는 각 테이블 셀을 가져옵니다. `accountnumber`를 엔터티 및 뷰에 적절 한 특성으로 바꿉니다.
```
$(document).ready(function (){
   $(".entitylist.entity-grid").on("loaded", function () {
      $(this).children(".view-grid").find("td[data-attribute='accountnumber']").each(function (i, e){
         var value = $(this).data(value);
         // now that you have the value you can do something to the value
      });
   });
});
```

이러한 설정은 엔터티 목록 양식의 구성 섹션에 있습니다. 기본적으로 **기본 설정만** 표시 됩니다. **고급 설정** 을 선택 하 여 추가 설정을 확인 합니다.

| CSS 클래스                | 결과 대화 상자에 적용 될 CSS 클래스를 지정 합니다.                                                            |
| 제목 CSS 클래스          | 결과 대화 상자의 제목 표시줄에 적용 될 CSS 클래스를 지정 합니다.                                                |
| 기본 단추 CSS 클래스 | 대화 상자의 기본 (삭제) 단추에 적용 되는 CSS 클래스를 지정 합니다.                                          |
| 닫기 단추 CSS 클래스   | 대화 상자의 닫기 (취소) 단추에 적용 되는 CSS 클래스를 지정 합니다.                                            |

## <a name="securing-entity-lists"></a>엔터티 목록 보안

엔터티 목록을 보호 하려면 레코드가 표시 되는 엔터티에 대 한 엔터티 권한을 구성 하 고 엔터티 목록 레코드의 **엔터티 권한 사용** 부울 값을 true로 설정 해야 합니다.

엔터티 목록을 보호 하는 작업은 해당 페이지에 액세스 하는 모든 사용자에 게 사용 권한이 부여 된 레코드만 표시 되도록 합니다. 이는 목록을 통해 표시 되는 모델 기반 앱 뷰에 추가 필터를 추가 하 여 수행 됩니다. 이 필터는 **읽기** 권한을 통해 사용자가 액세스할 수 있는 레코드에 대해서만 필터링 합니다.

또한 목록에 대해 정의 된 모든 작업은 레코드 별로 해당 작업에 대 한 해당 사용 권한을 적용 합니다. 즉, 레코드에 대 한 편집 권한이 있는 경우 해당 레코드에 대해 편집 작업이 활성화 됩니다. Delete, Create 등에도 마찬가지입니다. 사용할 수 있는 레코드가 없는 경우 목록이 로드 될 때이를 나타내는 메시지가 표시 됩니다.

그러나 적절 한 웹 사이트 디자인에서는 사용자가 엔터티에 대 한 사용 권한이 있는 역할에 속하지 않는 경우 (즉, 레코드가 표시 되어야 하는 상황이 발생 하지 않음), 페이지에 대 한 액세스 권한이 없어야 합니다. 웹 페이지 액세스 권한을 사용 하 여 페이지를 보호 하는 것이 가장 좋습니다.

## <a name="adding-a-view-details-page"></a>보기 세부 정보 페이지 추가

웹 페이지에 대 한 자세히 보기 조회에 대 한 웹 페이지를 설정 하면 관련 양식이 나 페이지의 구성에 따라 표에 나열 된 레코드의 세부 정보를 읽기 전용 또는 편집으로 볼 수 있습니다.

이 페이지는 완전히 사용자 지정 된 페이지 템플릿이 될 수 있으며,이는 액체를 사용 하 여 만들 수 있습니다. 가장 일반적인 시나리오는 세부 정보 페이지가 엔터티 양식이 나 웹 폼을 포함 하는 웹 페이지 일 것입니다.

유의 해야 할 중요 한 사항은 표에 나열 된 각 레코드가 세부 정보 페이지에 대 한 하이퍼링크를 포함 하 고 링크에는 레코드 ID가 있는 명명 된 쿼리 문자열 매개 변수가 포함 된다는 것입니다. 쿼리 문자열 매개 변수의 이름은 엔터티 목록에 지정 된 ID 쿼리 문자열 매개 변수 이름에 따라 달라 집니다. 최종 주의 사항은 대상 세부 정보 웹 페이지에서 해당 데이터를 쿼리하고 로드 하는 데 필요한 레코드의 ID를 가져오기 위해이 쿼리 문자열 매개 변수의 이름을 인식 해야 한다는 것입니다.

![보기 세부 정보 페이지 추가](../media/add-view-details-page.png "보기 세부 정보 페이지 추가")  

**엔터티 폼을 사용 하 여 세부 정보 표시**

엔터티 양식을 만들려면 [엔터티 양식](entity-forms.md) 페이지에 있는 지침을 참조 하세요.

엔터티 목록에서 레코드가 엔터티 형식으로 로드 되도록 하기 위해 알아야 하는 중요 한 설정은 다음과 같습니다.

엔터티 형식에 대 한 레코드 ID 쿼리 문자열 매개 변수 이름은 엔터티 목록의 ID 쿼리 문자열 매개 변수 이름과 일치 해야 합니다.

요구 사항에 따라 모드를 편집 또는 읽기 전용으로 사용할 수 있습니다.

**웹 폼을 사용 하 여 세부 정보 표시**

다음은 엔터티 목록의 레코드가 웹 폼에 로드 되도록 하기 위해 알아야 하는 중요 한 설정입니다.

웹 양식 단계의 기본 키 쿼리 문자열 매개 변수 이름은 엔터티 목록의 ID 쿼리 문자열 매개 변수 이름과 일치 해야 합니다.

요구 사항에 따라 모드를 편집 또는 읽기 전용으로 사용할 수 있습니다.

**Create 함수에 대 한 세부 정보 페이지 사용**

사용자 지정 페이지, 엔터티 양식 또는 웹 폼을 동일한 방식으로 Create 함수에 사용할 수 있습니다. 이는 폼에서 만들기 작업을 정의 하는 대신 사용할 수 있습니다. 만들기 작업 *및* 사용자 지정 작업 정의를 모두 정의할 수는 없습니다. 사용자 지정 작업 정의는 우선적으로 적용 됩니다.

엔터티 목록에서 조회 만들기에 웹 페이지를 할당 하 고 구성을 사용 하 여 만들기 작업을 지정 하지 않는 경우 목록에 만들기 단추가 렌더링 됩니다. 이 단추는 사용자를 만들기 위해 지정한 사용자 지정 페이지에 연결 합니다.

## <a name="entity-list-filter-configuration"></a>엔터티 목록 필터 구성

엔터티 목록에서 레코드를 필터링 하는 기능을 추가 하는 것은 간단 합니다. 필터링 옵션을 사용 하도록 설정한 후 사용자에 게 표시할 필터 유형을 하나 이상 선택 합니다. 사용자가 제공한 텍스트와 일치 하는 특성을 기준으로 필터링 하거나 일련의 옵션 중에서 선택할 수 있습니다. 고급 찾기를 사용 하 여 가정할 수 있는 거의 모든 유형의 필터를 디자인할 수도 있습니다.

**엔터티 목록 필터 사용**

메타 데이터 필터 섹션에서 사용 확인란을 선택 합니다. 그러면 표시 될 때 엔터티 목록에 필터 영역이 추가 됩니다. 하나 이상의 필터 유형을 정의할 때까지 상자는 빈 상태로 표시 됩니다.

방향 설정을 사용 하 여 엔터티 목록의 필터 영역을 렌더링 하는 방법을 정의할 수 있습니다. 기본값인 가로는 엔터티 목록 위의 필터 영역을 렌더링 합니다. 세로 방향은 필터 영역을 엔터티 목록의 왼쪽에 있는 상자로 렌더링 합니다.

![메타 데이터 필터 설정](../media/metadata-filter-settings.png "메타 데이터 필터 설정")  

**필터 유형**

|**필터 유형**      |**설명**                                                                                                                                                                                                                               |
|----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 텍스트 필터          | 텍스트 상자를 사용 하 여 지정 된 엔터티의 선택 된 특성에서 일치 하는 텍스트를 검색 하 여 엔터티 목록을 필터링 합니다.                                                                                                                               |
| 특성 필터 집합 | 각 확인란을 사용 하 여 엔터티 목록을 필터링 합니다. 각 확인란은 지정 된 엔터티의 특정 특성과 비교 하 여 해당 조건을 일치 시 키 려 고 시도 합니다.                                                                                           |
| 조회 집합           | 일련의 확인란을 사용 하 여 엔터티 목록을 필터링 합니다. 각 확인란은 지정 된 엔터티의 레코드와 관련 엔터티에 대 한 레코드 간의 관계를 나타냅니다.                                                                         |
| 범위 필터 집합     | 각 확인란은 두 조건 (예: 0 보다 크거나 같고 100 보다 작음)을 나타낼 수 있다는 점을 제외 하 고 특성 필터 집합과 유사 합니다.                                                                    |
| 동적 선택 목록 집합 | 특성 필터 집합에서 선택 목록 값을 선택 하는 것과 유사 합니다. 동적 선택 목록 집합에는 필터링 할 선택 목록 옵션을 지정할 필요가 없습니다. 대신 엔터티 목록이 로드 될 때 옵션의 전체 목록을 생성 합니다. |
| 동적 조회 집합   | 조회 집합과 유사 합니다. 동적 조회 집합을 사용할 경우 필터링 할 조회 옵션을 지정할 필요가 없습니다. 대신 엔터티 목록이 로드 될 때 옵션의 전체 목록을 생성 합니다.                                           |
| FetchXML 필터      | FetchXML 필터 조건을 사용 하 여 엔터티 목록을 필터링 합니다.                                                                                                                                                                                     |

**텍스트 필터**

텍스트 필터는 엔터티 목록 엔터티 형식의 특성에 연결 된 엔터티 목록 필터 영역에 텍스트 상자를 추가 합니다. 사용자가 필터를 적용할 때 엔터티 목록에는 선택한 특성에 값이 포함 되어 있는 레코드만 표시 됩니다.

텍스트 필터를 추가 하려면 **+ 텍스트 필터**를 선택 합니다.

![텍스트 필터 추가](../media/add-text-filter.png "텍스트 필터 추가")  

텍스트 필터는 다음 특성을 사용 합니다.

|**Name**     |**설명**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| 특성도    | 필터링 할 엔터티 목록에서 선택한 엔터티 형식에 대 한 특성의 이름입니다. *형식 문자열을 가진 특성만 텍스트 필터에 사용할 수 있습니다.*                                                                                   |
| 표시 이름 | 엔터티 목록이 표시 될 때 필터의 레이블을 재정의 합니다. 기본적으로이 속성은 선택한 특성의 이름으로 자동 설정 됩니다. |

**특성 필터 집합**

특성 필터 집합은 엔터티 목록에서 선택한 엔터티 형식의 단일 특성에 연결 하 여 엔터티 목록을 필터링 하는 일련의 옵션을 추가 합니다. 사용자가 필터를 적용할 때 엔터티 목록에는 선택한 옵션 중 하나 이상과 정확히 일치 하는 레코드만 표시 됩니다.

![특성 필터 설정](../media/set-attribute-filter.png "특성 필터 설정")

특성 필터 집합은 다음 특성을 사용 합니다.

|**Name**     |**설명**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| 특성도    | 필터링 할 엔터티 목록에서 선택한 엔터티 형식에 대 한 특성의 이름입니다. 문자열, BigInt, Decimal, Double, Integer, Money, Picklist, DateTime 및 Boolean 형식의 특성만 텍스트 필터에 사용할 수 있습니다.
|표시 이름 | 엔터티 목록이 표시 될 때 필터의 레이블을 재정의 합니다. 기본적으로이 속성은 선택한 특성의 이름으로 자동 설정 됩니다.
| 옵션 |      필터링 기준으로 사용할 수 있는 값의 컬렉션입니다. 자세한 내용은 아래를 참조 하세요.                                                                              |

**특성 필터 집합 옵션**

특성 필터 집합은 일반적으로 선택 목록 및 부울 특성을 제외한 여러 옵션을 포함할 수 있습니다. 부울 특성 필터 집합에는 하나의 true 옵션과 하나의 false 옵션&mdash;한 개 또는 두 개의 옵션만 있을 수 있습니다. 선택 목록 특성 필터 집합에는 선택 목록의 가능한 각 값에 대해 최대 하나의 옵션만 있을 수 있습니다.

옵션에는 다음과 같은 특성이 있습니다.

|**Name**     |**설명**                                                                                                                                                                                  |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 연산자     | 결과를 필터링 하는 데 사용 되는 비교 연산자입니다. 예를 들어 같음, 보다 작음 등입니다. 옵션에 대 한 연산자 목록은 필터에 대해 선택한 특성의 유형에 따라 달라 집니다. 예를 들어 숫자 (10 진수) 형식에는 보다 작거나 보다 큰 연산자가 포함 되는 반면 문자열 특성은 또는 Contains로 시작 하는 등의 연산자를 사용 합니다. 선택 목록 및 부울 연산자는 항상 같습니다.                                                                                                                                               |
| Value        | 이 필터 조건에 사용 되는 실제 값입니다.                                                                                                                                                 |
| 표시 이름 | 필터 상자에서이 옵션에 대 한 표시 이름을 재정의 합니다. 기본적으로이 값은 Value 특성과 동일한 값으로 설정 됩니다.                                                             |

**조회 집합**

| 옵션      | 필터링 기준으로 사용할 수 있는 값의 컬렉션입니다. 자세한 내용은 아래를 참조 하세요.                                                                                 |

**범위 필터 집합**

**범위 필터 집합 옵션**

범위 필터 집합에는 원하는 수의 옵션이 있을 수 있습니다. 각 옵션은 하나 또는 두 개의 subconditions를 사용 하 여 필터 조건을 생성 합니다 .이 둘은 조건이 true가 되려면 모두 충족 되어야 합니다.

옵션에는 다음과 같은 특성이 있습니다.

|**Name**              |**설명**                                                                                                                                                                                |

| 값 1               | 이 필터 조건에 사용 되는 첫 번째 값입니다.                                                                                                                                                |
| 연산자 2 (선택 사항) | 결과를 필터링 하는 데 사용 되는 두 번째 비교 연산자입니다 (예: Equals 및 Less). 옵션에 대 한 연산자 목록은 필터에 대해 선택한 특성의 유형에 따라 달라 집니다. 예를 들어 숫자 (10 진수) 형식에는 보다 작거나 보다 큰 연산자가 포함 되는 반면 문자열 특성은 또는 Contains로 시작 하는 등의 연산자를 사용 합니다. 선택 목록 및 부울 연산자는 항상 같습니다.                                                                                                                                             |
| 값 2 (선택 사항)    | 이 필터 조건에 사용 되는 두 번째 값입니다.                                                                                                                                               |
| 표시 이름          | 필터 상자에서이 옵션에 대 한 표시 이름을 재정의 합니다. 기본적으로이 값은 선택한 연산자 및 값에 따라 동적으로 설정 됩니다.                                         |

**동적 선택 목록 집합**

동적 선택 목록 집합은 지정 된 선택 목록 필드의 모든 값을 나타내는을 기준으로 필터링 하는 일련의 옵션을 추가 합니다. 이는 특성 필터 집합에서 선택 목록을 선택 하는 것과 다릅니다. 특성 필터 집합에서 사용자가 필터링 할 수 있도록 하는 옵션 집합을 지정 해야 합니다. 동적 선택 목록 집합에서 선택 목록 필드를 지정 하기만 하면 전체 옵션 집합이 자동으로 제공 됩니다. 더 많은 제어가 필요한 경우에는 특성 필터 집합을 사용 하는 것이 좋습니다.

**동적 조회 집합**

동적 조회 집합은 엔터티 목록에서 선택한 엔터티 형식에 대 한 관련 엔터티에 연결 하 여 엔터티 목록을 필터링 하는 동적 일련의 옵션을 추가 합니다. 사용자가 필터를 적용할 때 엔터티 목록에는 선택한 관련 레코드 중 하나 이상과 정확히 일치 하는 레코드만 표시 됩니다.

## <a name="entity-list-map-view"></a>엔터티 목록 지도 보기

엔터티 목록을 사용 하면 검색 기능을 통해 [!INCLUDE[pn-bing](../../../includes/pn-bing.md)] 지도에서 제공 하는 데이터의 지도 보기를 사용 하도록 설정 하 고 구성 하 여 주소 근처의 위치를 찾을 수 있습니다. 위도 및 경도 좌표 값으로 레코드를 채우고이 섹션에 나열 된 필수 구성 옵션을 지정 하 여 레코드를 지도에 압정로 렌더링할 수 있습니다. 위도 또는 경도 값이 없는 모든 레코드는 검색에서 제외 됩니다. 페이지의 초기 로드는 기본 중심 위도 및 기본 중심 경도 좌표에서 거리 값 필드 (지정 된 거리 단위에 따라 마일 또는 km)의 초기 값에 모든 레코드를 표시 합니다. 지도 보기가 사용 되는 경우 지정 된 뷰가 무시 되 고 데이터 집합에 거리 쿼리가 적용 되어 매핑 가능한 결과를 반환 합니다.
>[!Note] 
>이 옵션은 독일어 소 버린 클라우드 환경에서 지원 되지 않습니다. 지도 보기 섹션은이 환경에서 표시 되지 않습니다.


## <a name="entity-list-calendar-view"></a>엔터티 목록 일정 보기

엔터티 목록 달력 뷰를 사용 하 여 엔터티 목록을 일정으로 렌더링 합니다. 각 개별 레코드는 단일 이벤트로 작동 하도록 구성 됩니다.

달력을 사용 하 여 레코드를 표시 하려면 해당 레코드에 최소한의 날짜 필드를 포함 해야 합니다. 이벤트의 시작과 종료 시간이 정확한 경우 적절 한 필드를 준비 해야 합니다. 이러한 필드가 구성 된 것으로 가정 하면 엔터티 목록 달력 보기가 포털에 표시 됩니다.

## <a name="enhanced-view-filter-for-entity-lists"></a>엔터티 목록에 대 한 향상 된 뷰 필터

사용 하도록 설정 하면 엔터티를 OData 피드에 게시할 수 있습니다. OData 프로토콜은 RESTful 웹 서비스를 통해 데이터와 상호 작용 하기 위한 응용 프로그램 수준 프로토콜입니다. 이 피드의 데이터는 클라이언트 쪽 웹 응용 프로그램에서 사용 하거나 [!INCLUDE[pn-excel-short](../../../includes/pn-excel-short.md)]으로 가져올 수 있는 웹 브라우저에서 볼 수 있습니다.

## <a name="entity-list-odata-feeds"></a>엔터티 목록 OData 피드

레코드를 보호 하려는 경우 엔터티 권한을 사용할 수 있지만, 현재 포털 사용자와 관련 된 필터 옵션 집합의 일부로만 필터를 제공 하려는 경우에는 엔터티 목록 기능을 사용할 수 있습니다. 이 기능은 현재 사용자, 사용자의 부모 계정 또는 웹 사이트를 어느 수준에서 나 필터링 할 수 있도록 지원 합니다. 단일 연락처 레코드와 일치 하는 뷰 필터를 작성 하기만 하면 코드에서 필터 조건 섹션의 필드에 값을 할당할 필요 없이 런타임 시 값을 실제 값으로 대체&mdash;.

> [!Note]
> 게시 된 OData 피드는 익명 이며 권한 부여 검사를 포함 하지 않습니다. 따라서 익명 포털 액세스에 적합 하지 않은 데이터에 대해서는 oData 피드를 사용 하도록 설정 하지 않는 것이 중요 합니다.
