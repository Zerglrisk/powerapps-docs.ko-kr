# <a name="import-update-and-export-solutions"></a>솔루션 가져오기, 업데이트 및 내보내기 

 아래 단계를 사용하여 솔루션을 가져올 수 있습니다. 신뢰할 수 있는 원본에서 얻은 솔루션만 가져옵니다. 사용자 지정에는 데이터를 외부 원본으로 보낼 수 있는 코드가 포함될 수 있습니다. 기본 솔루션은 해당 솔루션을 내보냈던 조직에만 가져올 수 있으며 다른 조직에는 가져올 수 없습니다.  
  
1. **[설정](../model-driven-apps/advanced-navigation.md#settings)** > **솔루션**으로 이동합니다.  
  
2.  솔루션 목록 메뉴에서 **가져오기**를 선택합니다.  
  
3.  **솔루션 가져오기** 대화 상자의 **솔루션 패키지 선택** 단계에서 가져오려는 솔루션이 포함된 압축 파일(.zip 또는 .cab)을 찾습니다. 
  
4.  **다음**을 선택합니다.  
  
5.  **가져오기**를 선택하기 전에 솔루션에 대한 정보를 확인할 수 있습니다.  
  
6.  솔루션 가져오기가 완료되는 동안 몇 분 정도 기다려야 합니다. 성공한 경우 결과를 볼 수 있으며 **닫기**를 선택하면 됩니다.  
  
 게시가 필요한 변경 사항을 가져온 경우 사용자 지정을 게시해야 사용 가능한 상태가 됩니다. 
  
 가져오기가 실패한 경우 캡처된 오류나 경고를 보여 주는 보고서가 표시됩니다. **로그 파일 다운로드**를 선택하여 가져오기 실패 원인에 대한 세부 사항을 캡처할 수 있습니다. 솔루션 가져오기가 실패하는 가장 일반적인 원인은 솔루션에 필수 솔루션 구성 요소가 포함되어 있지 않다는 것입니다.  
  
 로그 파일을 다운로드하면 Office Excel을 사용하여 열고 내용을 볼 수 있는 XML 파일을 찾게 됩니다.  
  
> [!NOTE]
>  활성 회람 규칙 집합은 편집할 수 없습니다. 따라서 활성 회람 규칙 집합이 포함된 솔루션을 같은 ID로 규칙이 이미 있는 조직으로 가져오는 경우 솔루션 가져오기가 실패합니다. 자세한 정보: [자동으로 사례를 라우팅하는 규칙 만들기](https://docs.microsoft.com/dynamics365/customer-engagement/customer-service/create-rules-automatically-route-cases)

## <a name="update-solutions"></a>솔루션 업데이트  
 기존 관리 솔루션에 대한 업데이트를 설치하고자 할 수 있습니다. 이 절차는 몇 가지 다른 옵션을 제외하고는 새로운 관리 솔루션 설치와 유사합니다. 다른 사람으로부터 얻은 솔루션을 업데이트하는 경우 솔루션 게시자에게 어떤 옵션을 선택해야 하는지에 대한 지침을 받아야 합니다.  
  
1. **[설정](../model-driven-apps/advanced-navigation.md#settings)** > **솔루션**으로 이동합니다.   
  
2.  솔루션 목록 메뉴에서 **가져오기**를 선택합니다.  
  
3.  **솔루션 가져오기** 대화 상자의 **솔루션 패키지 선택** 단계에서 업데이트하려는 솔루션이 포함된 압축 파일(.zip 또는 .cab)을 찾습니다.  
4.  **다음**을 선택합니다.  
  
5.  **다음**을 선택하기 전에 솔루션에 대한 정보를 확인할 수 있습니다. 이 페이지에는 **이 솔루션 패키지에는 솔루션에 대해 이미 설치된 업데이트가 포함됩니다**라는 노란색 막대가 표시됩니다.  
  
6.  다음 옵션이 제공됩니다.  
  
    - **사용자 지정 유지 관리(권장)**  
  
         이 옵션을 선택하면 구성 요소에 대해 수행된 관리되지 않는 사용자 지정은 유지 관리되지만 이 솔루션에 포함된 일부 업데이트는 적용되지 않습니다.  
  
    - **사용자 지정 덮어쓰기**  
  
         이 옵션을 선택하면 이 솔루션에 포함된 구성 요소에 대해 이전에 수행된 관리되지 않는 모든 사용자 지정을 덮어씁니다. 이 솔루션에 포함된 모든 업데이트가 적용됩니다.  
  
     적절한 옵션을 선택한 후 **다음**을 선택합니다.  
  
7.  솔루션 가져오기가 완료되는 동안 몇 분 정도 기다려야 합니다. 성공한 경우 결과를 볼 수 있으며 **닫기**를 선택하면 됩니다.  
  
 게시가 필요한 변경 사항을 가져온 경우 사용자 지정을 게시해야 사용 가능한 상태가 됩니다. 
  
 솔루션 게시자는 기존의 관리되지 않는 사용자 지정을 내보내고 사용자 지정을 덮어쓰는 옵션을 사용하여 관리되는 솔루션을 업데이트한 다음, 관리되지 않는 사용자 지정을 다시 가져오도록 요청할 수 있습니다. 이렇게 하면 사용자 지정 내용을 유지하면서 예상되는 변경 내용을 적용할 수 있습니다.

## <a name="export-solutions"></a>솔루션 내보내기  
 관리되지 않는 사용자 지정을 주기적으로 내보내서 문제가 발생할 경우를 대비한 백업을 준비하는 것이 좋습니다. 관리 솔루션을 내보낼 수 없습니다.  
  
1. **[설정](../model-driven-apps/advanced-navigation.md#settings)** > **솔루션**으로 이동합니다.   
  
2.  목록에서 내보내려는 솔루션을 선택하고 **내보내기**를 선택합니다.  
  
3.  **사용자 지정 게시** 단계에서는 게시된 사용자 지정만 내보내고 **다음**을 선택하기 전에 **모든 사용자 지정 게시** 옵션을 사용할 수 있다는 알림이 제공됩니다.  
  
4.  솔루션에 누락된 필수 구성 요소가 포함되어 있으면 **누락된 필수 구성 요소** 단계가 표시됩니다. 관리되지 않는 솔루션을 원래 조직으로 다시 가져오려는 경우에만 이 경고를 무시할 수 있습니다. 그렇지 않은 경우 대화 상자의 지침에 따라 내보내기를 취소하고 필요한 구성요소를 추가합니다.  
  
5.  **시스템 설정 내보내기(고급)** 단계에서 솔루션에 포함할 특정 시스템 설정을 선택할 수 있습니다. 솔루션이 시스템 설정 그룹에 따라 다르면 해당 솔루션을 선택하고 **다음**을 선택합니다.  
  
     각 옵션과 함께 포함될 설정에 대한 자세한 내용은 아래 **솔루션 내보내기에 대한 옵션 설정**을 참조하세요.  
  
6.  **패키지 유형** 단계에서는 솔루션을 **관리되지 않는** 또는 **관리되는** 솔루션으로 내보낼지 여부를 선택해야 합니다.  
  
7.  다음 단계를 통해 특정 Dynamics 365 버전에 대한 대상 솔루션을 선택할 수 있습니다. 이 옵션은 일반적으로 이전 버전과 호환되는 솔루션을 내보내려는 ISV에 의해 사용됩니다. 이 솔루션을 사용 중인 조직 버전과 동일한 버전으로 업그레이드되지 않은 조직으로 가져오지 않는 경우 기본값을 적용합니다.   
  
8.  솔루션 파일을 다운로드하려면 **내보내기**를 선택합니다.  
  
 파일 다운로드에 대한 정확한 동작은 브라우저마다 다릅니다.
