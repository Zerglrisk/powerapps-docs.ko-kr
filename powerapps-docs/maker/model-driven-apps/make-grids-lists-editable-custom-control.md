# <a name="make-model-driven-app-grids-lists-editable-using-the-editable-grid-custom-control"></a>편집 가능한 그리드 사용자 지정 컨트롤을 사용하여 모델 기반 앱 그리드(목록)를 편집 가능하게 만들기

Dynamics CRM의 이전 릴리스에서는 사용자가 그리드(목록이라고도 함) 또는 양식의 하위 그리드에서 직접 데이터를 입력할 수 없었습니다. 그리드에서 레코드를 선택하여 양식을 열고 데이터를 편집한 후 저장하려면 여러 단계가 필요했습니다. 편집 가능한 그리드를 사용하면 웹앱, 태블릿 또는 휴대폰 사용과 관계없이 그리드와 하위 그리드에서 직접 다양한 인라인 편집을 할 수 있습니다.  
  
 ![편집 가능한 그리드 예제](media/editable-grid-examples.png "편집 가능한 그리드 예제")  
  
 편집 가능한 그리드 사용자 지정 컨트롤을 통해 편집 가능한 그리드를 사용하는 경우 사용자는 기본 조회 필드 및 옵션 집합을 포함하여 대부분의 필드 유형을 편집할 수 있습니다.  

**편집 가능한 그리드 지원:**
  
-   엔터티나 또는 하위 그리드 수준의 레코드를 인라인으로 편집(사용자 지정 엔터티 포함)  
  
-   시스템 보기 및 개인 보기  
  
-   웹 및 모바일 클라이언트  
  
-   키보드 또는 마우스를 사용한 탐색  
  
-   그룹화 및 정렬(현재 보기의 열을 기준으로 그룹화/정렬 가능)  
  
-   필터링  
  
-   열 이동 및 크기 조정  
  
-   페이지 매김  
  
-   그룹화, 정렬, 필터링, 페이지 매김 및 열 이동과 크기 조정을 위해 세션 간에 변경 내용 저장  
  
-   조회 구성  
  
-   계산된 필드 및 롤업 필드  
  
-   비즈니스 규칙(오류 메시지 표시, 필드 값 설정, 필요한 비즈니스 설정, 기본값 설정, 필드 잠금 또는 잠금 해제)  
  
-   JavaScript 이벤트  
  
-   보안 역할에 따른 셀 활성화 또는 비활성화  
  
-   사용자는 검색 및 차트를 계속 사용할 수 있으며 읽기 전용 그리드처럼 작업 표시줄에 액세스할 수 있습니다.  
  
## <a name="make-main-grids-editable"></a>주 그리드를 편집 가능하게 만들기  
  
1.  [솔루션 탐색기](advanced-navigation.md#solution-explorer)를 엽니다.  
  
2.  **엔터티** 목록에서 적절한 엔터티를 열고 **컨트롤** 탭을 선택한 다음, **컨트롤 추가**를 선택합니다.  
  
     ![편집 가능한 그리드 사용자 지정 컨트롤 추가](media/add-editable-grids-custom-control.png "편집 가능한 그리드 사용자 지정 컨트롤 추가")  
  
3.  **컨트롤 추가** 대화 상자에서 **편집 가능한 그리드**를 선택한 다음, **추가**를 선택합니다.  
  
4.  추가된 **편집 가능한 그리드** 행에서 그리드를 적용할 폼 팩터를 선택합니다. 이렇게 하면 편집 가능한 그리드 컨트롤이 선택한 폼 팩터에 대한 기본 컨트롤이 됩니다.  
  
     ![폼 팩터 선택과 편집 가능한 그리드 행](media/editable-grid-row-wit-factor-selection.png "폼 팩터 선택과 편집 가능한 그리드 행")    

   > [!NOTE]
   >  런타임 시 사용자는 편집 가능한 그리드와 읽기 전용 그리드 사이를 전환할 수 있습니다.  
      
5.  조회를 추가하려면 **편집 가능한 그리드** 옵션 그룹에서 **조회 추가**를 선택한 다음, **구성 속성 "조회 추가"** 대화 상자에서 다음을 선택합니다.  
  
    1.  **사용 가능한 보기** 목록에서 조회를 추가할 보기를 선택합니다(예: **내 활성 계정** 선택).  
  
    2.  **사용 가능한 열** 목록에서 추가할 조회 열을 선택합니다(예: **기본 연락처** 선택).  
  
    3.  **기본 보기** 목록에서 조회 필드의 데이터 원본을 선택합니다.  
  
    4.  표시되는 레코드를 제한하려면 **레코드만 표시 위치** 확인란을 선택한 다음, **확인**을 선택합니다.  
  
         ![편집 가능한 그리드 컨트롤에서 조회 추가](media/add-lookup-in-editable-grid-control.png "편집 가능한 그리드 컨트롤에서 조회 추가")  
     
6.  중첩된 그리드가 있는 경우 **중첩된 그리드 보기**에 대한 연필 단추를 선택한 다음, 중첩된 그리드에 대한 엔터티와 보기를 선택합니다. **중첩된 그리드 부모 ID**의 경우 엔터티에 대한 관계를 선택합니다. 예를 들어 ParentAccountID 필드는 **계정** 및 **연락처** 엔터티를 연결합니다.  
  
    > [!NOTE]
    >  중첩된 그리드는 웹이 아닌 휴대폰과 태블릿에서만 사용할 수 있습니다.  
  
7.  사용자가 보기의 모든 열로 데이터를 그룹화하지 못하도록 하려면(예: 공간을 절약하려는 경우) **열로 그룹화** 행에서 연필 단추를 선택한 다음, **구성 속성 "열로 그룹화"** 대화 상자에서 **사용 안 함**을 선택한 후, **확인**을 선택합니다.  
  
    > [!TIP]
    >  이는 주로 양식의 하위 그리드에 유용합니다.  
  
8.  JavaScript 이벤트를 추가하려면 **이벤트** 탭을 선택한 다음, 적절한 엔터티, 필드 및 이벤트를 선택합니다. 자세한 정보: [개발자 설명서: 편집 가능한 그리드 사용](/dynamics365/customer-engagement/developer/customize-dev/use-editable-grids-dynamics-365.md)  
  
     ![편집 가능한 그리드 컨트롤에서 이벤트 추가](media/add-events-in-editable-grid-control.png "편집 가능한 그리드 컨트롤에서 이벤트 추가")  
  
9. 작업을 저장하려면 작업 표시줄에서 **저장**을 선택합니다.  
  
10. 팀에 변경 내용을 적용할 준비가 되면 작업 표시줄에서 **게시**를 선택합니다.  
  
11. 변경 내용을 테스트하려면 5단계에서 지정한 보기로 이동한 다음, 인라인 편집 변경 내용을 적용합니다.  
  
## <a name="make-a-sub-grid-on-a-form-editable"></a>양식 편집 가능에서 하위 그리드 만들기

> [!NOTE] 
> - 편집 가능한 그리드 변경 내용을 하위 그리드 내에 저장하려면 사용자는 양식을 벗어나기 전에 명시적으로 저장해야 합니다.
> - 레거시 양식(Dynamics CRM 2016 이전 버전)을 사용하고 하위 그리드에서 편집 가능한 그리드를 사용하도록 설정하는 경우 편집 가능한 하위 그리드가 렌더링되지 않습니다. 시스템 관리자는 필요한 경우 시스템 설정에서 레거시 양식을 해제할 수 있습니다.
  
1.  [솔루션 탐색기](advanced-navigation.md#solution-explorer)를 엽니다.  
  
2.  하위 그리드가 포함된 양식을 엽니다.  
  
3.  적절한 컨트롤을 선택한 다음, 리본에서 **속성 변경**을 선택합니다.  
  
4.  **속성 설정** 대화 상자에서 **컨트롤**을 선택하고 **컨트롤 추가**를 선택한 다음, 위에 나열된 것과 동일한 단계를 수행합니다.

##  <a name="data-types-that-arent-editable-in-an-editable-grid"></a>편집 가능한 그리드에서 편집할 수 없는 데이터 형식
편집 가능한 그리드에서는 고객 및 Partylist 조회 필드; 복합(주소) 필드; 상태/상태 필드; 엔터티 관련 필드의 데이터 형식은 편집할 수 없습니다(예: 계정 엔터티에는 연락처 필드는 편집 가능하지만 EmailAdress(연락처) 필드는 편집할 수 없는 연락처 조회가 포함되어 있음).
