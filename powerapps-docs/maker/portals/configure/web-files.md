# <a name="create-and-manage-web-files"></a>웹 파일 만들기 및 관리

웹 파일은 포털 웹 사이트에서 이미지, 문서 및 기타 파일 형식을 저장 하는 데 사용 되는 다운로드 가능 파일을 나타냅니다.

포털에서는 지정 된 파일의 실제 내용을 저장 하기 위해 웹 파일 레코드와 연결 된 메모의 첨부 파일 기능을 사용 합니다. 웹 파일과 연결 된 최신 메모의 첨부 파일은 파일 내용으로 사용 됩니다. 따라서 포털에서 지원할 수 있는 웹 파일 콘텐츠의 크기는 Dynamics 365 설치에서 지 원하는 메모 첨부 파일 크기에 따라 결정 됩니다.

## <a name="manage-web-files"></a>웹 파일 관리

웹 파일은 PowerApps 포털 내에서 생성, 편집 및 삭제할 수 있습니다.

1. [포털 관리 앱](configure-portal.md)을 엽니다.

2. **포털** > **웹 파일**로 이동 합니다.

3. 새 웹 파일을 만들려면 **새로**만들기를 선택 합니다.

4. 기존 웹 파일을 편집 하려면 웹 파일 이름을 선택 합니다.

5. 필드에 적절 한 값을 입력 합니다.

6. **저장 후 닫기**를 선택합니다.

### <a name="web-file-attributes"></a>웹 파일 특성

아래 표에서는 포털에서 사용 되는 여러 표준 웹 파일 특성을 설명 합니다. 콘텐츠/표시 지향 특성의 대부분을 렌더링 하는 방법은 사용 된 페이지 템플릿 및 포털 개발자가 제어 하는 것입니다.

| 이름                | 설명               |
|---------------------|-----------------------|
|이름 |엔터티를 설명 하는 이름입니다. 이 값은 대부분의 템플릿 (예: 링크 타이틀)에서 파일 제목으로 사용 됩니다. 이 필드는 필수입니다.   |
|사이트나   |엔터티가 속한 웹 사이트입니다. 이 필드는 필수입니다.   |
|부모 페이지   |웹 사이트 콘텐츠 계층 구조에 있는 엔터티의 부모 웹 페이지입니다. <br>파일이 부모 페이지를 가질 필요는 없지만 예를 들어 파일에 부모 블로그 게시가 있는 경우가 있습니다. 대부분의 경우 부모 페이지를 제공 하는 것이 좋습니다.  |
|부분 URL   |이 페이지의 포털 URL을 작성 하는 데 사용 되는 URL 경로 세그먼트입니다. <br>웹 사이트의 단일 루트 (홈) 페이지 – 연결 된 부모 페이지가 없는 단일 페이지 –의 부분 URL 값은/여야 합니다.<br>부분 URL 값은 URL 경로 세그먼트로 사용 됩니다. 따라서?, #,!,%와 같은 잘못 된 URL 경로 문자를 포함 해서는 안 됩니다. Adxstudio 포털 Url은 슬래시 (/)와 부분 URL 값을 함께 조인 하 여 생성 되므로 일반적으로 슬래시를 포함 하지 않아야 합니다. 부분 URL 값을 문자, 숫자, 하이픈 또는 밑줄로 제한 하는 것이 좋습니다. 예: press-release, Site_Header.  |
|게시 상태   |파일이 사이트에 표시 되는지 여부를 지시할 수 있는 파일의 현재 게시 워크플로 상태입니다. 이 기능을 사용 하는 가장 일반적인 용도는 콘텐츠에 대 한 게시 된/초안 제어를 제공 하는 것입니다.<br>콘텐츠 관리 권한이 있는 사용자에 게 미리 보기 모드를 사용할 수 있는 권한이 부여 될 수 있으며,이를 통해 이러한 사용자는 게시 되지 않은 콘텐츠를 볼 수 있습니다 (미리 보기).   |
| 날짜 표시        | 이 특성은 표시 목적 으로만 템플릿에서 사용할 수 있는 날짜/시간 값입니다. 이는 기능에 영향을 주지 않지만 예를 들어, 보도 날짜를 보도 문서에 수동으로 지정 하는 등의 작업에 유용할 수 있습니다.    |
| 릴리스 날짜        | 파일이 포털에 표시 되는 날짜/시간을 제어 합니다. 현재 날짜/시간이이 날짜 보다 이전 이면이 파일은 표시 되지 않습니다. 이에 대 한 예외는 콘텐츠 관리 권한이 있는 사용자에 게 미리 보기 모드를 사용할 수 있는 권한이 부여 될 수 있기 때문입니다 .이를 통해 이러한 사용자는 릴리스되지 않은 콘텐츠를 볼 수 있습니다 (미리 보기). 이는 뉴스 또는 보도와 같이 시간이 중요 한 콘텐츠의 릴리스를 제어 하는 데 유용 합니다. |
| 만료 날짜     | 파일이 포털에 표시 되기 전 까지의 날짜/시간을 제어 합니다. 현재 날짜/시간이이 날짜 이후 이면이 파일이 표시 되지 않습니다. 이에 대 한 예외는 콘텐츠 관리 권한이 있는 사용자에 게 미리 보기 모드를 사용할 수 있는 권한이 부여 될 수 있기 때문입니다. 그러면 이러한 사용자가 만료 된 콘텐츠를 볼 수 있습니다 (미리 보기).                |
| 정리             | 파일에 대 한 간단한 설명입니다 .이 값은 일반적으로 파일에 대 한 링크를 렌더링 하는 포털 탐색 요소에 파일에 대 한 설명을 추가 하는 데 사용 됩니다.      |
| 사이트 맵에서 숨김 | 파일이 표시 되는지 여부를 제어 하는 포털 사이트 맵 이 값을 선택 하면 해당 파일은 사이트의 URL에서 계속 사용할 수 있으며에 연결할 수 있지만 표준 탐색 요소 (메뉴 등)는 페이지를 포함 하지 않습니다.      |
| 표시 순서       | 부모 페이지가 동일한 다른 엔터티를 기준으로 파일이 배치 되는 순서를 나타내는 정수 값입니다. 이는 예를 들어 지정 된 페이지의 자식 엔터티에 대 한 링크 목록이 포털에서 렌더링 될 때 파일 및 기타 사이트 맵 엔터티의 순서를 제어 합니다.      |
| 클라우드 Blob 주소  | `<container>/<filename>`형식의 텍스트 값으로,이 파일의 내용이 Azure Blob Storage에 저장 됨을 나타냅니다.        |
| 콘텐츠 처리 | 인라인 또는 첨부 옵션입니다. Inline을 지정 하는 경우 브라우저는 브라우저 창에서 렌더링을 시도 해야 합니다. 그렇지 않으면 사용자에 게 파일을 다운로드 하거나 열도록 요청 하는 메시지가 표시 됩니다. 첨부 파일을 지정 하는 경우 파일을 다운로드 하거나 열도록 사용자에 게 즉시 표시 하 고, 브라우저에서 파일을 로드 하지 않을 지 여부에 관계 없이 파일을 로드 하지 않습니다.                                                                                        |
| 추적 사용     | 사용 하도록 설정 하면이 웹 파일에 대 한 모든 요청이 기록 됩니다. 사용자가 인증 된 경우 날짜 & 시간, IP 주소 및 연락처 레코드를 사용 하 여 웹 파일 로그 레코드가 생성 됩니다.      |
