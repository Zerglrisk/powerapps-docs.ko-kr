# <a name="tutorial-add-or-edit-model-driven-app-components-in-the-powerapps-app-designer"></a>자습서: PowerApps 앱 디자이너에서 모델 기반 앱 구성 요소 추가 또는 편집

이 자습서에서는 모델 기반 앱에서 구성 요소를 추가하고 제거하는 방법에 대해 알아봅니다. 

모델 기반 앱은 다양한 구성 요소로 구성됩니다. 앱에 두 가지 유형의 구성 요소인 아티팩트 및 엔터티 자산을 추가할 수 있습니다. 앱 디자이너의 컨텍스트에서 엔터티, 대시보드 및 비즈니스 프로세스 흐름은 모두 앱의 아티팩트입니다. 엔터티 자산은 양식, 보기, 차트 및 대시보드로 구성됩니다.  
  
앱 디자이너는 기본 솔루션의 기존 메타데이터를 참조합니다. 이를 사용하여 양식, 보기, 차트 및 대시보드와 같은 구성 요소를 만들 수 있습니다.  
  
## <a name="app-designer-layout"></a>앱 디자이너 레이아웃  
 앱 디자이너에는 두 개의 주요 영역이 있습니다. 왼쪽에는 앱 구성 요소를 추가하는 캔버스가 있습니다.  
  
![앱 디자이너 캔버스](../model-driven-apps/media/app-designer-canvas-pane.png)

 오른쪽에는 구성 요소를 선택하고 구성 요소 속성을 설정하는 데 사용할 탭이 있습니다.  
  
 ![앱 디자이너 구성 요소](../model-driven-apps/media/app-designer-canvas-components-tab.png "앱 디자이너 구성 요소")  
  
 캔버스에서 사이트 맵, 비즈니스 프로세스 흐름, 대시보드 및 엔터티에 대한 영역이 표시됩니다. 대시보드 또는 비즈니스 프로세스 흐름을 선택하거나 사이트 맵을 구성하면 앱 디자이너는 이러한 구성 요소에서 사용되는 엔터티를 캔버스에 자동으로 추가합니다. 엔터티를 배치한 후 각 엔터티를 선택하고 양식, 보기 및 차트와 같은 필수 엔터티 자산을 추가하면 됩니다.
 
 **캔버스 검색**을 사용하여 캔버스에서 구성 요소를 검색할 수도 있습니다. **캔버스 검색**을 선택하면 가장 오른쪽 창에 있는 탭의 오른쪽에 새 검색 탭이 열립니다.   
  
 ![캔버스 검색 옵션](media/app-designer-search-tab.png "캔버스 검색")

## <a name="open-an-app"></a>앱 열기
1. [PowerApps](https://web.powerapps.com/)에 로그인합니다. 

2. **모델 기반** > **앱**을 선택한 다음, 기존 앱을 선택하거나 **앱 만들기**를 선택합니다. 앱을 만드는 방법에 대한 자세한 내용은 [앱 디자이너를 사용하여 모델 기반 앱 만들기 또는 편집](create-edit-app.md#create-an-app)을 참조하세요.

## <a name="add-an-artifact-entity-dashboard-or-business-process-flow"></a>아티팩트(엔터티, 대시보드 또는 비즈니스 프로세스 흐름) 추가  
 앱에 대시보드 또는 비즈니스 프로세스 흐름을 추가하면 엔터티가 앱에 자동으로 추가됩니다. 엔터티를 추가하면 해당 자산의 타일이 자동으로 추가됩니다. 디자이너 캔버스에 아티팩트를 추가하는 방법에는 명령 모음에서 ![디자이너의 추가 단추](../model-driven-apps/media/dynamics365-designer-addbutton.PNG "디자이너의 추가 단추")인 **추가** 단추를 사용하거나 **구성 요소** 탭에서 타일을 사용하는 두 가지가 있습니다.  
  
 다음은 앱에 대시보드를 추가하는 단계입니다. 동일한 단계를 사용하여 비즈니스 프로세스 흐름 또는 엔터티를 추가합니다.  
  
1.  앱 디자이너 캔버스에서 **대시보드** 타일을 선택합니다.  
  
     앱 디자이너 캔버스의 오른쪽 창에는 기본 솔루션에서 사용할 수 있는 대시보드가 표시됩니다.  
  
    > [!TIP]
    >  또는 다음 중 하나를 수행할 수도 있습니다.  
    >   
    > - ![디자이너의 추가 단추](../model-driven-apps/media/dynamics365-designer-addbutton.PNG "디자이너의 추가 단추") **추가**를 선택한 다음, **대시보드**를 선택합니다.  
    > - **구성 요소** 탭의 **아티팩트** 아래에서 **대시보드**를 선택합니다.  
  
2.  **검색** 상자에 원하는 대시보드 이름에 대한 몇 가지 키워드를 입력합니다.  
  
     대시보드 목록이 필터링되어 키워드와 일치하는 결과를 표시합니다.  
  
3.  사용자가 선택한 대시보드만 사용하도록 하려면 추가할 대시보드에 대한 확인란을 선택합니다. 다음 유형의 대시보드 중에서 선택할 수 있습니다.
    - **클래식 대시보드**는 웹앱과 통합 인터페이스 앱 모두에 나타납니다.
    - **대화형 대시보드**는 통합 인터페이스 앱에만 나타납니다. 앱의 클라이언트 유형을 웹앱으로 선택한 경우 **대화형 대시보드** 옵션이 표시되지 않습니다.

     해당 대시보드는 앱 디자이너 캔버스의 **대시보드** 타일에 추가됩니다. **대시보드** 타일에도 앱에 추가한 대시보드 수가 표시됩니다. 대시보드를 선택하지 않으면 대시보드 수 대신 **모두**가 나타나고 사용자가 앱을 사용할 때 모든 대시모드를 사용할 수 있습니다.  
  
     대시보드에서 사용하는 모든 엔터티도 **엔터티 보기** 영역에 추가됩니다. 예를 들어 고객 서비스 관리자 대시보드를 추가하면 사례, 자격 및 큐 항목 엔터티가 엔터티 보기 영역에 추가됩니다. 각 엔터티에 대해 해당 자산의 타일도 추가됩니다. 이러한 타일을 사용하여 양식, 보기 및 차트를 추가할 수 있습니다. 자세한 내용: [PowerApps 앱 디자이너에서 앱 구성 요소 추가 또는 편집](add-edit-app-components.md#bkmk_AddEntityAssets)   
  
    ![앱 디자이너 캔버스에 엔터티 추가](../model-driven-apps/media/add-entity-app-designer-canvas.png "앱 디자이너 캔버스에 엔터티 추가")  
  
4.  원하는 대시보드가 기본 솔루션에 없는 경우 캔버스 오른쪽에 있는 **구성 요소** 탭에서 **새로 만들기**를 선택하여 대시보드를 만듭니다.  
  
     ![앱 디자이너의 구성 요소 탭에서 새 링크 만들기](../model-driven-apps/media/app-designer-components-tab-create-new.png "앱 디자이너의 구성 요소 탭에서 새 링크 만들기")  
  
     대시보드 디자이너가 열립니다. 자세한 내용: [대시보드 만들기 및 편집](create-edit-dashboards.md)  
  
    > [!NOTE]
    > - 비즈니스 프로세스 흐름 또는 엔터티를 추가할 때 **새로 만들기** 옵션이 해당 디자이너를 엽니다. 비즈니스 프로세스 흐름 또는 엔터티 만들기에 대한 자세한 내용은 [비즈니스 프로세스 흐름 만들기](/flow/create-business-process-flow) 및 [사용자 지정 엔터티 만들기](https://docs.microsoft.com/powerapps/maker/common-data-service/data-platform-create-entity)를 참조하세요.  
      
  
5.  아티팩트를 추가한 후 명령 모음에서 **저장**을 선택합니다.  
  
<a name="bkmk_AddEntityAssets"></a>   
## <a name="add-entity-assets-forms-views-charts-or-dashboards"></a>엔터티 자산(양식, 보기, 차트 또는 대시보드) 추가  
 아티팩트를 배치하면 양식, 보기, 차트 및 대시보드와 같은 엔터티 자산을 앱에 추가할 수 있습니다.
또한 통합 인터페이스 클라이언트를 사용하는 경우 엔터티 대시보드 자산도 앱에 추가할 수 있습니다.  
  
 이 섹션에서는 앱에 양식을 추가하는 단계를 설명합니다. 동일한 단계를 사용하여 앱에 보기 또는 차트를 추가합니다.  
  
1.  앱 디자이너 캔버스에서 양식을 추가할 엔터티의 **양식** 타일을 선택합니다.  
  
     앱 디자이너 캔버스에서 엔터티의 전체 행이 선택됩니다. 오른쪽에는 선택한 엔터티에 대한 모든 기존 양식이 표시됩니다.  
  
    > [!NOTE]
    >  또는 다음 중 하나를 수행할 수도 있습니다.  
    >   
    > - ![디자이너의 추가 단추](../model-driven-apps/media/dynamics365-designer-addbutton.PNG "디자이너의 추가 단추") **추가**를 선택한 다음, **양식**을 선택합니다.  
    > - **구성 요소** 탭의 **엔터티 자산** 아래에서 **양식**을 선택합니다.  
  
    > [!TIP]
    >  앱용으로 선택하는 모든 엔터티의 **추가 옵션** 단추(**...**)는 **구성 요소** 탭의 **엔터티 선택** 목록에 나타납니다. 선택한 엔터티에 대한 모든 자산을 추가하려면 **추가 옵션**(**...** )을 선택한 다음, **모든 자산 추가**를 선택합니다.  
  
2.  앱 사용자가 선택한 양식만 사용하도록 하려면 추가할 양식에 대한 확인란을 선택합니다. 양식은 사용자가 앱에서 데이터를 보고 상호 작용하는 방법을 정의합니다. 
 
     선택한 엔터티의 양식 타일에 추가된 양식 수가 표시됩니다.  
  
     ![사례 엔터티에 대한 양식 타일](../model-driven-apps/media/add-forms-entity.png "사례 엔터티에 대한 양식 타일")  
  
     예를 들어 엔터티에 대한 양식을 선택하지 않으면 해당 엔터티에 대한 모든 양식은 앱을 사용하는 동안 최종 사용자에게 표시됩니다. 이 동작은 보기 또는 차트를 선택하지 않는 경우에도 보기 및 차트와 유사합니다. 이렇게 하면 사용 가능한 모든 구성 요소로 작업해야 할 때 신속하게 앱을 만들 수 있습니다. 앱 디자인 중에 각 구성 요소를 선택할 필요가 없습니다.  

     대시보드 또는 비즈니스 프로세스 흐름을 선택하지 않으면 사용자가 앱을 사용하는 동안 모든 대시보드 및 비즈니스 프로세스 흐름을 사용할 수 있습니다.
  
    > [!NOTE]
    > 앱을 실행하려면 추가하는 각 엔터티에 하나 이상의 활성 양식이 있어야 합니다. 여러 양식을 선택한 경우 사용자가 앱을 실행할 때 기본 솔루션에 표시되는 첫 번째 활성 양식이 사용됩니다.  
  
3.  목록에서 사용할 수 없는 새 양식을 추가하려면 **새로 만들기**를 선택합니다.  
  
     드롭다운 목록에서 만들려는 양식 유형을 선택합니다.  
  
    > [!NOTE]
    >  드롭다운 목록은 양식을 추가할 때만 사용할 수 있습니다. 보기 및 차트에 사용할 수 없습니다.  
  
     양식 디자이너가 열립니다. 자세한 내용: [양식 만들기 및 디자인](create-design-forms.md)  
  
     보기 또는 차트를 추가할 때 **새로 만들기** 옵션이 해당 디자이너를 엽니다. 자세한 정보: [보기 이해](create-edit-views.md) 및 [시스템 차트 만들기 또는 편집](create-edit-system-chart.md)  
  
    > [!NOTE]
    >  보기를 추가할 때 솔루션 탐색기의 **보기** 노드 아래에 나열된 공용 보기만 참조할 수 있습니다.  
  
4. 아래쪽 화살표 ![드롭다운 아이콘](../model-driven-apps/media/drop-down-icon.png "아래쪽 화살표")를 선택하여 타일을 확장하고 추가된 양식 목록을 참조하세요.  
  
     ![앱 디자이너에서 양식 타일 확장](../model-driven-apps/media/app-designer-expanded-form-tile.png "앱 디자이너에서 양식 타일 확장")  
  
5.  이 단계를 반복하여 앱에 엔터티 보기 및 차트를 추가합니다.  
  
6.  **저장**을 선택합니다.  
  
## <a name="edit-or-remove-artifacts"></a>아티팩트 편집 또는 제거  
  
- 대시보드 또는 비즈니스 프로세스 흐름을 편집하려면 아래쪽 화살표 ![드롭다운 아이콘](../model-driven-apps/media/drop-down-icon.png "아래쪽 화살표")를 선택하여 타일을 확장한 다음, 대시보드 또는 편집할 비즈니스 프로세스 흐름에 해당하는 사이트 맵 디자이너 단추 ![사이트 맵 디자이너 열기 단추](../model-driven-apps/media/dynamics365-open-designer.PNG "사이트 맵 디자이너 열기 단추")를 선택합니다.  
  
     선택한 아티팩트의 디자이너가 열립니다.  
  
- 대시보드 또는 비즈니스 프로세스 흐름을 제거하려면 아래쪽 화살표 ![드롭다운 아이콘](../model-driven-apps/media/drop-down-icon.png "아래쪽 화살표")를 선택하여 타일을 확장한 다음, 제거할 대시보드 또는 비즈니스 프로세스 흐름을 선택합니다. 명령 모음에서 **제거**를 선택합니다.  

    대시보드 또는 비즈니스 프로세스 흐름을 제거하는 또 다른 방법은 **구성 요소** 탭에서 해당 확인란을 선택 취소하는 것입니다.
  
- 엔터티를 편집하거나 제거하려면 엔터티 타일을 선택한 다음, 명령 모음에서 **편집** 또는 **제거**를 선택합니다. 엔터티를 편집하면 엔터티를 변경할 수 있는 솔루션 탐색기가 열립니다.  
  
     구성 요소를 제거하는 또 다른 방법은 대시보드, 비즈니스 프로세스 흐름 또는 엔터티 타일을 선택하는 것입니다. **구성 요소** 탭의 디자이너에서 제거할 아티팩트의 확인란을 선택 취소합니다.  
  
    > [!NOTE]
    >  &mdash;엔터티의 표시 이름 또는 설명 변경과 같이&mdash; 엔터티를 변경하는 경우 솔루션 탐색기에 변경 내용이 게시되지 않는 한 변경 내용이 앱 디자이너에 표시되지 않습니다.  
  
## <a name="edit-or-remove-entity-assets"></a>엔터티 자산 편집 또는 제거  

### <a name="edit-entity-assets"></a>엔터티 자산 편집
  
1. 아래쪽 화살표 ![드롭다운 아이콘](../model-driven-apps/media/drop-down-icon.png "아래쪽 화살표")를 선택하여 양식, 보기, 차트 또는 대시보드의 타일을 확장합니다.  
  
2. 편집할 양식, 보기, 차트 또는 대시보드를 선택합니다.  
  
3. 명령 모음에서 **편집**을 선택합니다.

   또는

   양식, 보기, 차트 및 대시보드에 해당하는 사이트 맵 디자이너 단추 ![사이트 맵 디자이너 열기 단추](../model-driven-apps/media/dynamics365-open-designer.PNG "사이트 맵 디자이너 열기 단추")를 선택합니다.  

### <a name="remove-entity-assets"></a>엔터티 자산 제거  

1. 아래쪽 화살표 ![드롭다운 아이콘](../model-driven-apps/media/drop-down-icon.png "아래쪽 화살표")를 선택하여 양식, 보기, 차트 또는 대시보드의 타일을 확장합니다.  
  
2. 편집할 양식, 보기, 차트 또는 대시보드를 선택합니다.

3. 명령 모음에서 **제거**를 선택합니다. 

또는 양식, 보기, 차트 또는 대시보드를 선택한 다음, **구성 요소** 탭의 디자이너에서 제거할 자산의 확인란을 선택 취소합니다.
