# <a name="powerapps-community-plan-a-free-development-environment-for-individual-use"></a>PowerApps 커뮤니티 플랜: 개별 사용을 위한 무료 개발 환경
기술을 구축하고 Powerapps, Microsoft Flow 및 Common Data Service에 대한 자세한 내용을 보려면 PowerApps 커뮤니티 플랜이 적합 한 플랜입니다. PowerApps 커뮤니티 플랜은 다음을 수행할 수 있는 개별 사용을 위한 무료 개발 환경을 제공합니다.

* PowerApps 및 Microsoft Flow의 모든 기능을 사용하여 비즈니스 앱 및 워크플로를 구축하는 방법을 배웁니다.
* 100개 이상의 [기본 제공 커넥터](./canvas-apps/connections-list.md)를 사용하거나 [사용자 지정 커넥터](./canvas-apps/register-custom-api.md)를 만들어 데이터 원본에 연결합니다.
* [Common Data Service](https://docs.microsoft.com/common-data-service/entity-reference/introduction)를 사용하여 일반적인 데이터 모델 및 [SDK](https://aka.ms/eek20s)를 사용하여 강력한 비즈니스 앱을 빌드하는 방법을 살펴봅니다.
* 개별 환경에서 만든 솔루션을 내보내고 이를 [AppSource에 나열](./canvas-apps/dev-appsource-test-drive.md)하여 고객이 테스트 드라이브를 할 수 있도록 합니다.

## <a name="who-can-sign-up-for-the-powerapps-community-plan"></a>PowerApps 커뮤니티 플랜에 등록할 수 있는 사람은 누구 입니까?
[회사 또는 학교 계정](signup-for-powerapps.md#faq)이 있는 사용자는 PowerApps 커뮤니티 플랜에 등록할 수 있습니다. 그러나 특히 다음과 같은 경우 이 플랜을 추천합니다.

* 기술을 구축하고 PowerApps, Microsoft Flow 및 Common Data Service에 대해 자세히 알아보려는 경우.
* AppSource에 배포할 비즈니스 앱 및 워크플로를 구축하는 데 관심이 있는 경우.

## <a name="where-can-i-sign-up-for-the-powerapps-community-plan"></a>PowerApps 커뮤니티 플랜은 어디에서 등록할 수 있습니까?
[PowerApps 커뮤니티 플랜 웹 사이트](https://powerapps.microsoft.com/communityplan)에서 등록합니다. Office 365 또는 Dynamics 365를 사용하는 PowerApps의 기존 사용자는 [개별 사용을 위한 환경을 생성](https://web.powerapps.com/community/signup)할 수도 있습니다.

커뮤니티 플랜에 등록한 후에는 [PowerApps 사이트](https://web.powerapps.com?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc) 리디렉션되고 개별 환경을 이용하게 됩니다. 환경 이름은 'John Doe의 환경'과 같이 사용자 이름으로 지정됩니다. 이미 해당 이름을 가진 환경이 있는 경우 개별 환경은 ' John Doe(1)의 환경'으로 이름이 지정됩니다.  다음 이미지는 환경이 어떻게 나타나는지 보여줍니다.

![커뮤니티 플랜을 위한 개별 환경](./media/dev-community-plan/individual-environment.png)

### <a name="get-the-community-plan-with-visual-studio-dev-essentials"></a>Visual Studio Dev Essentials을 사용하여 커뮤니티 플랜 얻기
Visual Studio Dev Essentials 사용자인 경우에는 PowerApps가 혜택에 포함됩니다. [내 혜택](https://my.visualstudio.com/benefits)을 방문하고 PowerApps 타일을 클릭하거나 탭하여 PowerApps 커뮤니티 플랜에 등록합니다.

![Visual Studio의 커뮤니티 플랜](./media/dev-community-plan/visual-studio.png)

## <a name="which-features-are-included-in-the-powerapps-community-plan"></a>PowerApps 커뮤니티 플랜에 포함되는 기능은 무엇입니까?
개별 환경을 사용하면 다음과 같은 기능을 얻을 수 있습니다.

| **기능** | **개별 사용을 위한 환경** |
| --- | --- |
| **주요 기능** | |
| 앱 만들기 및 실행 |예. 무제한 앱을 만들 수 있습니다. |
| 앱 공유\* |아니오 |
| Common Data Service 사용 |예** |
| Common Data Service를 사용하여 데이터 모델링 |예** |
| 환경 및 사용자 정책에 대한 엔터프라이즈급 관리 |지원 |
| **연결** | |
| Office 365, Dynamics 365 및 기타 커넥터에 연결 |지원 |
| Azure SQL, Dropbox, Twitter 등과 같은 클라우드 기반 서비스에 연결 |지원 |
| Salesforce, DB2 등과 같은 프리미엄 커넥터 사용 |지원 |
| 온-프레미스 게이트웨이를 사용하여 온-프레미스 데이터에 액세스 |지원 |
| 사용자 지정 커넥터를 만들어 자신의 시스템에 연결 |예. 무제한 사용자 지정 커넥터를 만들 수 있습니다. |
| **Common Data Service** | |
| Common Data Service에서 응용 프로그램 만들기 및 실행 |예** |
| Common Data Service에서 데이터 모델링 |예** |
| Common Data Service에서 데이터베이스 만들기 |예** |
| **관리** | |
| 동료를 환경 제작자 및 관리자로 추가 |아니오 |
| 데이터베이스 역할에 동료 추가 |아니오 |
| Office 365 관리자가 설정한 데이터 정책을 지원 |지원 |
| 개별 환경에 대한 데이터 정책 설정 |지원 |

*테넌트의 다른 사용자와 앱, 흐름, 연결 등을 공유할 수 없습니다. 또한 다른 사용자를 환경 관리자나 제작자로 추가하거나 관리 센터에서 데이터베이스 역할로 추가할 수 없습니다.

\*\*Common Data Service를 사용하여 데이터베이스 만들기는 현재 개별 환경에서 사용할 수 없으며 곧 다시 사용할 수 있게 됩니다. 이전에 데이터베이스를 만든 경우에는 문제 없이 계속 사용할 수 있습니다. 

## <a name="what-are-the-capacity-limits-for-the-individual-environment"></a>개별 환경에 대한 생산 능력 제한은 무엇입니까?

| **생산 능력** |  |
| --- | --- |
| 흐름 실행/월 |750 |
| 데이터베이스 크기 |200MB |
| 파일 저장소 |2GB |

우리가 포함하는 수량에 추가 기능을 적용할 수 없습니다. 생산 능력 제한에 도달하면 PowerApps 플랜 2를 구입하는 것이 좋습니다. [PowerApps 가격 책정 페이지](https://powerapps.microsoft.com/pricing/)에서 자세히 알아보세요.

> [!NOTE]
> 사용 여부에 관계 없이 개별 환경의 생산 능력은 회사의 전체 할당량에 영향을 주지 않습니다.

## <a name="publishing-to-appsource"></a>AppSource에 게시
고객과 공유하고 싶은 앱이 있습니까? 이제 고객과의 앱 및 흐름을 공유하고 비즈니스를 위한 잠재 고객을 창출하는 방법으로 [AppSource.com](https://appsource.microsoft.com)에서 PowerApps 테스트 드라이브 솔루션을 지원합니다. 자세한 내용은 [고객이 AppSource에서 앱을 테스트 드라이브하도록 유도](./canvas-apps/dev-appsource-test-drive.md)를 참조하십시오.

## <a name="frequently-asked-questions"></a>질문과 대답
**Q:** 환경의 생산 능력 한계에 도달하면 어떻게 해야 합니까?

**A:** 이 환경은 팀 또는 프로덕션 사용이 아닌 개인 용도로 사용하기 위한 것이므로 제공되는 생산 능력이 제한됩니다. 제공되는 생산 능력은 다음과 같습니다.

| **생산 능력** |  |
| --- | --- |
| 흐름 실행/월 |750 |
| 데이터베이스 크기 |200MB |
| 파일 저장소 |2GB |

하나 이상의 생산 능력 제한에 도달하는 경우 프로덕션 사용을 지원하는 플랜을 구입하는 것이 좋습니다. [PowerApps 가격 책정 페이지](https://powerapps.microsoft.com/pricing/)에서 플랜 및 제한 사항에 대해 자세히 알아보십시오.

**Q:** 개별 환경에서 만든 앱, 흐름 및 기타 리소스를 다른 환경으로 전송할 수 있습니까?

**A:** 예. 이 환경에서 다른 환경으로 리소스를 내보낼 수 있어야 합니다. 자세한 내용은 [환경 및 테넌트 앱 마이그레이션](../administrator/environment-and-tenant-migration.md)을 참조하십시오.

**Q:** 내 PowerApps 커뮤니티 플랜 구독이 만료됩니까?

**A:** PowerApps 커뮤니티 플랜 구독은 영구히 무료로 사용할 수 있습니다. 개별 환경을 적극적으로 사용하는 경우 해당 환경의 리소스나 기능에 액세스할 수 없게 됩니다. 그러나 오랜 시간 동안 비활성 상태에서 처음으로 Common Data Service 데이터베이스에 액세스할 때 지연이 발생할 수 있습니다.  이 지연은 Common Data Service에 저장된 데이터나 엔터티에 영향을 주지 않습니다.

**Q:** 여러 개별 환경을 가져오거나 만들 수 있습니까?

**A:** 아니요, 커뮤니티 플랜에 등록할 때 PowerApps에서 생성되는 하나의 개별 환경만 사용할 수 있습니다.

**Q:** PowerApps 플랜 2 평가판과 PowerApps 커뮤니티 플랜의 차이점은 무엇입니까? 그리고 어느 것에 등록해야 합니까?

**A:** PowerApps 플랜 2 평가판과 PowerApps 커뮤니티 플랜 모두 무료이지만 다른 목적으로 만들어집니다.

**PowerApps 플랜 2 평가판**은 30일 동안 PowerApps 플랜 2를 제공합니다. 이는 PowerApps, Common Data Service 및 Microsoft Flow를 사용해 보기 위한 것입니다. 평가판 기간이 만료되면 플랜을 구입할 수 있습니다. 이미 Office 365 또는 Dynamics 365와 함께 PowerApps를 사용하는 경우 PowerApps 플랜 2에서 사용할 수 있는 PowerApps의 프리미엄 기능을 시험해 볼 수 있는 적절한 플랜입니다.

**PowerApps 커뮤니티 플랜**을 사용하면 개별 사용을 위해 PowerApps 프리미엄 기능, Common Data Service 및 Microsoft Flow에 액세스할 수 있습니다. 이 플랜은 주로 학습 목적이거나 [AppSource 테스트 드라이브](./canvas-apps/dev-appsource-test-drive.md)를 위해 배포될 비즈니스 솔루션을 만드는 것을 의미합니다. 이 플랜은 영구히 사용할 수 있지만 PowerApps, Common Data Services 및 Microsoft Flow에 대한 기술을 학습하고 구축하는 데만 사용 됩니다.

**Q:** 내 개인 계정으로 등록할 수 있습니까?

**A:** 아니요. [회사 또는 학교 계정](signup-for-powerapps.md#faq)으로만 등록할 수 있습니다. 현재 개인 계정으로는 등록을 지원하지 않습니다.

**Q:** 내 개별 환경을 삭제할 수 있습니까?

**A:** 이 환경은 직접 삭제할 수 없습니다. 테넌트 관리자에게 환경을 삭제할 수 있는 권한이 있습니다.
