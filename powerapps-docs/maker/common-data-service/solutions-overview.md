# <a name="solutions-overview"></a>솔루션 개요  

 솔루션은 모델 기반 앱을 구입, 공유하거나 그렇지 않은 경우 한 조직에서 다른 조직으로 전송할 수 있도록 하기 위해 존재합니다. [AppSource](https://appsource.microsoft.com/) 또는 ISV(독립 소프트웨어 공급업체)로부터 솔루션을 가져올 수 있습니다. 솔루션은 환경에 앱으로 가져오거나 기존 앱에 일련의 사용자 지정을 적용할 수 있는 파일입니다.  
  
자세한 정보: [Whitepaper: Patterns and Principles for Solution Builders(백서: 솔루션 작성기에 대한 패턴 및 원칙)](http://go.microsoft.com/fwlink/p/?LinkID=533946)  
  
> [!NOTE]
>  배포할 앱을 만드는 ISV라면 솔루션을 사용해야 합니다. 솔루션 사용에 대한 자세한 내용은 [솔루션을 사용하여 확장 패키지 및 배포](https://msdn.microsoft.com/library/gg334530.aspx)를 참조하세요.  
  
 단지 조직용 또는 Dynamics 365 사용자 지정용 PowerApps 앱을 만드는 데 관심이 있는 사용자라면 솔루션에 대해 알아야 할 내용은 다음과 같습니다.  
  
-   솔루션 만들기는 선택 사항입니다. 솔루션을 만들지 않고도 PowerApps 환경에서 앱을 직접 빌드하거나 사용자 지정할 수 있습니다.  
  
-   PowerApps 환경을 직접 사용자 지정하는 경우 **Common Data Services 기본 솔루션**이라는 특수 솔루션으로 작업합니다. 이 솔루션에는 시스템의 모든 구성 요소가 들어 있습니다.  
  
-   기본 솔루션을 내보내 조직에서 정의한 사용자 지정의 백업을 만들 수 있습니다. 이것은 최악의 시나리오에 적합합니다.  
  
<a name="BKMK_SolutionComponents"></a>   
### <a name="solution-components"></a>솔루션 구성 요소  
 솔루션 구성 요소는 잠재적으로 사용자 지정할 수 있는 항목을 나타냅니다. 솔루션에 포함할 수 있는 모든 것은 솔루션 구성 요소입니다. 다음은 솔루션에서 볼 수 있는 솔루션 구성 요소의 목록입니다.  
  
-   응용 프로그램 리본  

-   앱 
  
-   문서 템플릿  
  
-   비즈니스 규칙  
  
-   차트  
  
-   연결 역할  
  
-   계약 템플릿  
 
-   사용자 지정 컨트롤
  
-   대시보드  
  
-   이메일 템플릿  
  
-   엔터티  
  
-   엔터티 관계  
  
-   필드  
  
-   필드 보안 프로필  
  
-   폼  
  
-   편지 병합 템플릿  
  
-   메시지  
  
-   옵션 집합  
  
-   플러그 인 어셈블리  
  
-   프로세스  
  
-   SDK 메시지 처리 단계  
  
-   보안 역할  
  
-   서비스 엔드포인트  
  
-   사이트 맵  

-   가상 엔터티 데이터 공급자

-   가상 엔터티 데이터 원본
  
-   웹 리소스  
  
 대부분의 솔루션 구성 요소는 다른 솔루션 구성 요소 내에 중첩되어 있습니다. 예를 들어 엔터티에는 양식, 보기, 차트, 필드, 엔터티 관계, 메시지 및 비즈니스 규칙이 포함됩니다. 이러한 각 솔루션 구성 요소에는 엔터티가 존재해야 합니다. 필드는 엔터티 외부에 존재할 수 없습니다. 이것을 필드가 엔터티에 종속적이라고 말합니다. 앞의 목록에서 볼 수 있듯이 실제로는 두 가지 유형의 솔루션 구성 요소가 있지만 대부분 응용 프로그램에 표시되지 않습니다.  
  
 솔루션 구성 요소를 포함하는 목적은 관리 속성 및 모든 솔루션 종속성을 사용하여 사용자 지정할 수 있는 항목에 대한 제한 사항을 추적하여 아무 것도 남기지 않고 내보내고, 가져오며, 관리형 솔루션에서 삭제할 수 있도록 하는 것입니다.  
  
<a name="BKMK_ManagedAndUnmanagedSolutions"></a>   
### <a name="managed-and-unmanaged-solutions"></a>관리형 및 비관리형 솔루션  
 **관리형** 솔루션은 가져온 후에 제거할 수 있습니다. 솔루션을 제거하면 해당 솔루션의 모든 구성 요소가 제거됩니다.  
  
 **비관리형** 솔루션을 가져올 때 해당 솔루션의 모든 구성 요소를 기본 솔루션에 추가합니다. 솔루션을 제거하여 구성 요소를 제거할 수 없습니다.  
  
 이미 사용자 지정한 솔루션 구성 요소가 포함된 **비관리형** 솔루션을 가져오면 원본 사용자 지정을 비관리형 솔루션의 사용자 지정으로 덮어쓰게 됩니다. 이 작업을 실행 취소할 수 없습니다.  
  
> [!IMPORTANT]
>  모든 구성 요소를 기본 솔루션에 추가하고 기존 사용자 지정을 덮어쓰려는 경우에만 비관리형 솔루션을 설치합니다.  
  
 솔루션을 배포할 계획이 없더라도 비관리형 솔루션을 만들고 사용하여 사용자 지정한 응용 프로그램의 해당 부분만 포함하는 별도의 보기를 원할 수 있습니다. 항목을 사용자 지정할 때마다 자신이 만든 비관리형 솔루션에 추가하세요.  
  
 기본 솔루션은 비관리형 솔루션으로만 내보낼 수 있습니다.  
  
 **관리형** 솔루션을 만들려면 솔루션을 내보낼 때 관리형 솔루션 옵션을 선택합니다. 관리형 솔루션을 만드는 경우 해당 솔루션을 만들기 위해 사용한 것과 동일한 조직으로 다시 가져올 수 없습니다. 다른 조직으로만 가져올 수 있습니다.  
  
<a name="BKMK_HowSolutionsAreApplied"></a>   
### <a name="how-solutions-are-applied"></a>솔루션을 적용하는 방법  
 모든 솔루션은 레이어로 평가되어 앱이 실제로 수행할 작업을 결정합니다. 다음 다이어그램은 관리형 솔루션과 비관리형 솔루션이 평가되는 방식과 변경 내용이 조직에 표시되는 방식을 보여 줍니다.  
  
 ![솔루션 레이어](media/solution-layering.png "솔루션 레이어")  
  
 하단에서 시작하여 위쪽으로 작업:  
  
 **시스템 솔루션**  
 시스템 솔루션은 모든 조직에 있는 관리형 솔루션과 같습니다. 시스템 솔루션은 시스템의 모든 기본 구성 요소의 정의입니다.  
  
 **관리형 솔루션**  
 관리형 솔루션은 시스템 솔루션 구성 요소를 수정하고 새 구성 요소를 추가할 수 있습니다. 여러 관리형 솔루션이 설치되어 있는 경우 첫 번째 설치된 솔루션은 나중에 설치된 관리형 솔루션 아래에 있습니다. 즉, 설치된 두 번째 솔루션은 이전에 설치된 솔루션을 사용자 지정할 수 있습니다. 두 개의 관리형 솔루션이 충돌하는 정의를 포함할 때 일반적인 규칙은 “마지막 것이 우선”합니다. 관리형 솔루션을 제거하면 그 아래의 관리형 솔루션이 적용됩니다. 모든 관리형 솔루션을 제거하면 시스템 솔루션 내에 정의된 기본 동작이 적용됩니다.  
  
 **비관리형 사용자 지정**  
 비관리형 사용자 지정이란 비관리형 솔루션을 통해 조직에 적용한 모든 변경 사항을 말합니다. 시스템 솔루션은 관리 속성을 사용하여 사용자 지정하거나 사용자 지정할 수 없는 항목을 정의합니다. 관리형 솔루션의 게시자는 솔루션에 추가하는 솔루션 구성 요소를 사용자 지정하는 능력을 제한하는 능력과 동일한 기능을 갖추고 있습니다. 사용자 지정을 차단하는 관리 속성이 없는 모든 솔루션 구성 요소를 사용자 지정할 수 있습니다.  
  
 **응용 프로그램 동작**  
 이것은 조직에서 실제로 확인할 수 있는 것입니다. 기본 시스템 솔루션과 모든 관리형 솔루션 및 사용자가 적용한 비관리형 사용자 지정입니다.  
  
<a name="BKMK_ManagedProperties"></a>   
### <a name="managed-properties"></a>관리 속성  
 일부 구성 요소는 사용자 지정할 수 없습니다. 시스템 솔루션의 이러한 구성 요소에는 사용자 지정을 차단하는 메타데이터가 있습니다. 이것을 **관리 속성**이라고 합니다. 또한 관리형 솔루션의 게시자는 관리 속성을 설정하여 자신이 원하지 않는 방식으로 솔루션을 사용자 지정하지 못하도록 할 수 있습니다.  
  
<a name="BKMK_Dependencies"></a>   
### <a name="solution-dependencies"></a>솔루션 종속성  
 관리형 솔루션이 계층화된 방식으로 인해 일부 관리형 솔루션은 다른 관리형 솔루션의 솔루션 구성 요소에 종속될 수 있습니다. 일부 솔루션 게시자는 이를 활용하여 모듈식 솔루션을 빌드합니다. “기본” 관리형 솔루션을 먼저 설치한 다음, 기본 관리형 솔루션의 구성 요소를 추가로 사용자 지정하는 두 번째 관리형 설치할 수 있습니다. 두 번째 관리형 솔루션은 첫 번째 솔루션의 일부인 솔루션 구성 요소에 따라 다릅니다.  
  
 시스템은 솔루션 간에 이러한 종속성을 추적합니다. 설치되지 않은 기본 솔루션이 필요한 솔루션을 설치하려고 하면 해당 솔루션을 설치할 수 없습니다. 해당 솔루션을 사용하려면 다른 솔루션을 먼저 설치해야 한다는 메시지가 나타납니다. 마찬가지로 종속성으로 인해, 종속된 솔루션이 아직 설치된 상태에서는 기본 솔루션을 제거할 수 없습니다. 종속 솔루션을 먼저 제거한 후 기본 솔루션을 제거해야 합니다.
