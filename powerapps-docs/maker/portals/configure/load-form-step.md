# <a name="define-a-load-form-and-load-tab-step-type"></a>로드 양식 및 로드 탭 단계 타입 정의

단계 유형은 기본적으로 전체 웹 양식 프로세스 내에서 웹 양식 유형이 엔터티 양식으로 동작하도록 합니다. 엔터티 양식에서 가능한 옵션들과 유사한 양식을 불러옵니다.

## <a name="settings"></a>설정

| 이름                                  | 설명                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|---------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 이름                                  | 레코드를 설명하는 이름입니다. 필수 참석자                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 엔터티 이름                           | 양식이 로드되는 엔터티의 이름입니다. 필수 참석자                                                                                                                                                                                                                                                                                                                                                                                                             |
| 양식 이름                             | 렌더링할 대상 엔터티에 대한 양식의 이름입니다. 필수 참석자                                                                                                                                                                                                                                                                                                                                                                                                           |
| 탭 이름                              | 렌더링할 지정된 엔터티에 대한 양식의 탭 이름입니다. 선택 참석자                                                                                                                                                                                                                                                                                                                                                                                                  |
| 모드                                  | 다음 값 중 하나:<br><ul><li>삽입</li><li>편집</li><li>읽기 전용</li></ul>삽입을 선택하면 제출 시 양식에 새로운 레코드가 삽입됩니다. 편집을 지정하면 양식에서 기존 레코드를 편집할 수 있습니다. 읽기 전용을 선택하면 양식에 기존 레코드가 편집할 수 없는 형태로 나타납니다. 편집 및 읽기 전용을 사용하려면 원본 레코드가 존재해야 하며 포털에서 양식을 불러올 때 적절한 레코드를 선택할 수 있도록 '레코드 원본 종류'와 '레코드 ID 쿼리 문자열 매개 변수 이름' 필드에 매개 변수가 지정되어야 합니다.  |
| 탭에서 자동으로 단계 생성         | 확인되면 엔터티 양식의 여러 탭이 첫 번째 탭부터 시작하여 레코드가 삽입되는 마지막 제출로 이동할 때까지의 순차적인 단계로 표시됩니다. 기본적으로 확인 안 됨이 지정되어 있습니다. 확인 안 됨 값은 현재 단계에 대해 단 한 개의 탭 혹은 양식만이 렌더링됨을 의미합니다. 탭 이름이 지정되지 않으면 첫 번째 탭이 표시됩니다.                                              |
| 레코드 원본 종류                    | 다음 값 중 하나:<br><ul><li>쿼리 문자열</li><li>현재 포털 사용자</li><li>이전 단계의 결과</li></ul>쿼리 문자열을 선택하려면 URL의 쿼리 문자열의 매개 변수 이름이 양식에 제공되어야 합니다. '레코드 ID 쿼리 문자열 매개 변수 이름' 필드에서 지정할 수 있습니다. 현재 포털 사용자를 선택하면 현재 인증된 사용자에 대한 포털 사용자 레코드를 검색할 수 있습니다.<br>이전 단계의 결과를 선택하면 이전 단계의 웹 양식에 대한 레코드 원본이었던 레코드를 검색할 수 있습니다.                                                                                                                                                                                                                                                                                                                                                          |
| 레코드 ID 쿼리 문자열 매개 변수 이름 | 이 엔터티 양식을 포함하는 웹 페이지로 연결하는 URL의 쿼리 문자열에 제공된 매개 변수 이름입니다.                                                                                                                                                                                                                                                                                                                                                                                |
| 관계 이름                     | 레코드 원본 종류가 현재 포털 사용자의 레코드와 연결된 경우 필요합니다. 현재 포털 사용자 레코드와 대상 레코드의 관계의 논리 이름입니다. 엔터티 이름 필드에 지정한 것과 동일한 엔터티 유형을 반환해야 합니다.                                                                                                                                                                                                                           |
| Null인 경우 만들기 허용                  | 레코드 원본 형식이 현재 포털 사용자의 레코드와 연결된 경우 부울 값을 선택할 수 있습니다. 확인되면 관련 레코드가 존재하지 않을 경우, 사용자가 처음으로 만드는 것을 허용합니다. 혹은 레코드가 필요한 데이터 바인딩하는 양식으로서 존재하지 않는 경우에 예외가 표시됩니다.                                                                                                                                        |
| 엔터티 권한 사용             | 양식이 엔터티 권한을 존중하게 됩니다. 이전 버전과의 호환성 문제로 기본값은 false입니다. true로 설정되는 경우 양식에 액세스하려는 모든 사용자에게 명시적 권한이 요구됩니다. 이는 양식의 처음 단계에만 적용됩니다.                                                                                                                                                                                                                     |
||

## <a name="associate-the-current-portal-user-with-the-creation-of-a-record"></a>레코드 생성 시 현재 포털 사용자 연결

이 옵션은 포털 UI를 통해 레코드를 생성하는 포털 연락처를 추적하는 데 사용됩니다.

| 이름                                       | 설명                                                                                                                                                                             |
|--------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 현재 포털 사용자 연결              | 확인하면 현재 로그인한 사용자의 레코드가 대상 엔터티 레코드와 연결됩니다.                                                                             |
| 대상 엔터티 포털 사용자 조회 특성 | 포털 사용자를 저장하는 대상 엔터티의 특성의 논리적 이름입니다.                                                                                                     |
| 활동 당사자임                          | 대상 엔터티 포털 사용자 조회 특성이 활동 당사자 유형인지 여부를 나타내는 부울 값입니다. [ActivityParty 엔터티](https://msdn.microsoft.com/library/gg328549.aspx)를 참고하십시오. |  
||

## <a name="entity-reference"></a>엔터티 참조

다음 매개 변수는 양식을 저장할 때 엔터티의 참조에 대한 설정과 관련 있습니다.

양식에 의해 생성되거나 업데이트되는 현재 레코드를 다른 대상 레코드와 연결할 수 있습니다. 여러 엔터티 유형과 여러 단계가 있을 때 결과 레코드를 연결하고자 하거나 페이지가 연결하고자 하는 레코드 ID의 쿼리 문자열을 지나쳤을 때 유용합니다. 예를 들어, 일자리 게시물이 나열된 채용 정보 페이지가 있다고 합시다. 각 게시물에는 일자리 게시물의 ID가 담긴 입사지원서 양식으로의 링크가 있어 지원서를 생성했을 때 일자리 게시물이 레코드와 연결됩니다.

| 이름                                 | 설명                                                                                                                                                    |
|---------------------------------|------------|
| 저장 단계에서 엔터티 참조 설정         | 예 혹은 아니요. 예 값은 양식을 저장할 때 엔터티 참조를 할당해야 함을 나타냅니다. 그렇지 않으면 없음으로 설정됩니다.                                                                                                                 |
| 관계 이름                    | 두 엔터티 유형에 대해 주어진 관계에 대한 관계 정의 이름입니다.<br>**참고**: 대상 조회 특성 논리 이름을 지정하는 경우, 관계 이름을 지정하지 마십시오. 
| 엔터티 논리 이름                  | 참조 엔터티의 논리적 이름입니다.                |
| 대상 조회 특성 논리 이름 | 생성되거나 업데이트되는 대상 엔터티에 대한 조회 특성의 논리적 이름입니다. <br>**참고**: 대상 조회 특성 논리 이름을 지정하는 경우, 관계 이름을 지정하지 마십시오.                                                                           |
| 조회 필드 채우기                | 참조 엔터티에 대한 조회가 양식에 있을 경우 이 값을 확인하면 아래의 설정을 사용하여 검색한 값으로 양식의 필드가 채워집니다.                                                                                                                                                                                                                                                                               |
| 원본 유형                          | 다음 값 중 하나:<ul><li>쿼리 문자열</li><li>현재 포털 사용자</li><li>이전 단계의 결과</li></ul>쿼리 문자열을 선택하려면 URL의 쿼리 문자열의 매개 변수 이름이 양식에 제공되어야 합니다. **쿼리 문자열 이름** 필드에서 지정할 수 있습니다. 매개 변수가 기본 키라면 **쿼리 문자열이 기본 키입니다**에 대해 [예]를 선택합니다. 그렇지 않으면 [아니요]를 선택하고, **쿼리 특성 논리적 이름** 필드를 지정하여 쿼리에 대상 엔터티 특성의 논리 이름을 제공합니다. 현재 포털 사용자를 선택하면 현재 승인된 사용자의 연락처 레코드를 가져올 수 있습니다. 이전 단계의 결과를 선택하면 현재 단계 이전 단계의 결과로 생성된 레코드 또는 엔터티 원본 단계와 연관된 단계에 기반하여 생성된 특정 단계의 레코드를 가져올 수 있습니다.  |
| 참조 엔터티 단계                | 현재 단계에 대한 레코드와 연결하기 위해 이전 단계에서 생성되거나 편집된 엔터티를 가져오는 이전 단계의 웹 양식 단계 레코드입니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 쿼리 문자열 이름                    | 웹 양식을 포함하는 웹 페이지로 연결하는 URL의 쿼리 문자열에 제공된 매개 변수 이름입니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 쿼리 문자열은 기본 키          | '예'는 쿼리 문자열 값이 주요 키 값이라는 것을 나타냅니다. '아니요'는 쿼리 문자열 값이 주요 키가 아닌 다른 특성 형식임을 나타냅니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 쿼리 특성 논리 이름         | 레코드를 쿼리하는 특성의 논리적 이름입니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 읽기 전용 정보 표시                | 확인하면 페이지의 상단에 참조 레코드와 관련된 읽기 전용 정보를 표시하는 양식이 렌더링됩니다. 양식 이름이 요구됩니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 양식 이름                            | 읽기 전용 정보를 표시하는 데 사용되는 참조 엔터티에 대한 양식의 이름입니다.                                                      |
||
