# <a name="configure-notes-for-entity-forms-and-web-forms-on-portals"></a>포털에서 엔터티 양식 및 웹 양식에 대한 메모 구성

하위 표의 경우처럼 포털에서 관리되는 양식에 메모를 추가하는 것은 쉽습니다. [양식 디자이너](../model-driven-apps/create-design-forms.md)를 통해 모델 기반 앱 양식에 메모 컨트롤을 추가하기만 하면 됩니다. 메타데이터를 사용하여 메모 컨트롤의 동작을 구성할 수 있습니다.

> [!Note]                                                           
> 포털에 표시할 메모에 명시적 [엔터티 권한](configure/assign-entity-permissions.md)이 필요합니다. 읽기 및 편집을 위해 읽기 및 쓰기 권한이 부여되어야 합니다. 생성을 위한 두 권한이 존재해야 하며, 메모(주석) 엔터티를 위한 생성 및 첨부 권한이 부여되어야 하고, 메모가 첨부되고 있는 엔터티 타입에 두 번째 권한이 지정되고 첨부 권한이 부여되어야 합니다. 엔터티 권한을 적용하려면 해당 엔터티 양식 또는 웹 양식 단계에서 **엔터티 권한 사용** 선택란을 선택해야 합니다.

## <a name="notes-configuration-for-entity-forms"></a>엔터티 양식에 대한 메모 구성

1. [포털 관리 앱](configure/configure-portal.md)을 엽니다.
2. **포털** > **콘텐츠** > **엔터티 양식**으로 이동합니다. 활성 엔터티 양식 목록이 표시됩니다.
3. 메모 구성을 추가할 엔터티 양식을 선택합니다.
4. 상단 드롭다운 목록 또는 작업하고 있는 엔터티 양식 레코드의 기본 양식에 있는 하위 표를 사용하여 **엔터티 양식 메타데이터**로 이동합니다.
5. **새 엔터티 양식 메타데이터 추가**를 선택하여 새 레코드를 추가합니다.
6. **유형** 드롭다운 목록에서 **메모**를 선택합니다. 메모 구성별 설정이 표시됩니다. 대부분의 설정은 기본적으로 축소되어 있습니다. 섹션을 확장하여 추가 설정을 볼 수 있습니다.
7. 적절한 값을 입력하여 필드를 채웁니다. [!include[](../../includes/proc-more-information.md)] [특성](#attributes), [대화 옵션 만들기](#create-dialog-options), [대화 옵션 편집](#edit-dialog-options), [대화 옵션 삭제](#delete-dialog-options)
8. 양식을 저장합니다.

    ![엔터티 양식에 메모 구성 추가](media/add-note-configuration.png "엔터티 양식에 메모 구성 추가")  

    구성을 추가하면 노트 컨트롤이 포털에 활성화된 해당 옵션을 사용하여 렌더링될 것입니다.

| 메모 개인 정보 레이블    | 메모가 비공개임을 표시하는 레이블을 무효화합니다.                                                                                                         |
| 로드 메시지       | 메모 목록을 로드하고 있는 동안 표시되는 메시지를 무효화합니다.                                                                                              |
| 오류 메시지         | 메모 목록을 로드하려고 하는 동안 오류가 발생할 때 표시되는 메시지를 무효화합니다.                                                                     |
| 액세스 거부 메시지 | 사용자가 메모 목록을 보기에 충분한 권한을 갖고 있지 않을 때 표시되는 메시지를 무효화합니다.                                                    |
| 비어 있음 메시지         | 현재의 엔터티가 볼 수 있는 메모를 갖고 있지 않을 때 표시되는 메시지를 무효화합니다.                                                              |
| 목록 순서           | 메모가 표시될 순서를 설정할 수 있습니다. 주문 목록 설정을 사용하면 다음을 설정할 수 있습니다. <ul><li>특성: 원하는 정렬 기준 열의 논리적 이름</li><li>별칭: 쿼리에 있는 특성의 별칭</li><li>방향: 오름차순(가장 작은 것에서 가장 큰 것으로, 또는 처음에서 마지막으로) 또는 내림차순(가장 큰 것에서 가장 작은 것으로, 또는 마지막에서 처음으로)</li></ul> ![주문 목록 특성 설정](media/set-attributes-list-orders.png "S목록 주문의 속성 설정") 정렬 규칙을 추가하려면 "열"(4)을 선택하여 세부 정보를 입력합니다. 목록 순서는 최고 우선순위 목록의 상단으로부터 순서대로 처리될 것입니다.|
